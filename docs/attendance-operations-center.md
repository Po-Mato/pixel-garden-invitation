# 참석 현황 통합 운영 센터

관리자 화면 `?admin=rsvp`에서 개인 초대 명단, 발송·열람 상태, RSVP 응답과 참석 인원을 한 화면에서 관리한다.

## 집계 기준

- 초대 링크와 RSVP는 서버가 저장한 `rsvpId`로만 연결한다.
- 이름과 신부·신랑 측이 같아도 자동으로 합치지 않는다.
- 전체 참석 인원은 개인 링크 연결 여부와 관계없이 저장된 RSVP를 기준으로 집계한다.
- 관계 그룹 집계는 개인 초대 링크에 입력한 측과 그룹을 기준으로 한다.
- 어린이 인원은 전체 참석 인원을 넘을 수 없으며, 성인 인원은 `전체 참석 인원 - 어린이 인원`이다.
- 식사 인원은 참석 확정이면서 식사 예정인 RSVP의 전체 참석 인원이다.

연결되지 않은 RSVP, 삭제된 RSVP를 가리키는 링크, 이름·측 불일치와 중복 후보는 `명단 확인 필요`에 표시한다.

## 운영 흐름

1. 상단 요약에서 RSVP 수, 총 참석·성인·어린이·식사 인원과 후속 연락 필요 수를 확인한다.
2. 초대 진행 현황에서 발송, 열람, 응답 전환을 확인한다.
3. 측별·관계 그룹 집계로 테이블과 식사 준비 인원을 확인한다.
4. 후속 연락 목록을 미발송, 미열람, 미응답, 참석 미정, 연락 완료로 필터링한다.
5. 개인 링크 원문이 현재 탭에 있으면 재안내 문구를 복사하고, 실제 전달 후 발송 이력을 기록한다.
6. 전화나 대면 확인을 마치면 연락 완료로 표시한다. 필요하면 완료 표시를 취소할 수 있다.

개인 링크 원문은 서버에서 복원하지 않는다. 현재 관리자 탭에 토큰이 없으면 초대 링크 관리 화면에서 링크를 재발급해야 한다.

## CSV와 수정

- `운영 명단 CSV`는 초대·발송·열람·응답·인원·후속 연락·점검 상태를 함께 내보낸다.
- CSV는 UTF-8 BOM과 수식 실행 방지 처리를 사용하며 개인 링크 토큰을 포함하지 않는다.
- `답변 CSV`는 기존 RSVP 원본 목록을 내려받는다.
- 답변 수정에서 전체 참석 인원, 어린이 인원, 식사 여부와 전달사항을 관리한다.

## 데이터 보관

어린이 인원은 RSVP에, 후속 연락 완료 시각은 개인 초대 링크에 저장한다. 두 데이터는 기존 RSVP 보관 정책을 따르며 현재 일정에서는 예식 30일 후인 2027년 5월 31일에 자동 삭제된다.
