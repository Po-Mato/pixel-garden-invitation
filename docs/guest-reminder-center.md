# 하객 리마인드·재발송 운영 센터

## 접속

- 관리자 화면: `?admin=reminders`
- 참석 현황, 초대 링크, 공지·FAQ 관리자 화면의 `리마인드` 메뉴에서도 이동할 수 있습니다.
- 기존 관리자 비밀번호와 세션을 그대로 사용합니다.

## 권장 운영 일정

센터는 예식 시작일을 기준으로 30일 전, 14일 전, 7일 전, 하루 전 대상을 자동 분류합니다.

- 30일 전: 아직 RSVP를 제출하지 않은 하객
- 14일 전: 미열람·미응답·참석 미정 하객
- 7일 전: 답변 마감 전 미응답·참석 미정 하객
- 하루 전: 참석 또는 참석 미정으로 답변한 하객
- 수동 재안내: 활성 상태인 하객을 관리자가 직접 선택

같은 단계의 발송 이력이 이미 있거나 후속 연락 완료 상태인 하객은 `완료`로 구분됩니다. RSVP를 완료한 하객은 30일·14일·7일 전 재안내 대상에서 자동으로 제외됩니다.

## 재발송 절차

1. 권장 발송 일정에서 단계를 선택합니다.
2. 측, 관계 그룹, 이름, 처리 상태로 대상을 좁힙니다.
3. 개인 링크가 준비된 하객은 문구 복사, 시스템 공유, QR을 사용할 수 있습니다.
4. 개인 링크가 없는 하객은 `링크 준비`를 선택합니다. 재발급하면 기존 링크는 폐기됩니다.
5. 실제로 카카오톡·문자·직접 전달을 마친 뒤 `발송 완료 기록`을 저장합니다.

여러 명을 선택하면 개인 이름과 링크가 포함된 문구를 한 번에 복사할 수 있습니다. 실제 메시지 발송은 유료 문자 API 없이 휴대전화의 공유창이나 문구 복사 방식으로 진행합니다.

## 개인정보와 링크 보안

- 서버에는 개인 초대 링크 원문을 저장하지 않습니다.
- 관리자 브라우저 세션에서 생성하거나 재발급한 링크만 복사·공유·QR 생성에 사용할 수 있습니다.
- 리마인드 이력에는 하객 연락처나 메시지 본문을 저장하지 않습니다.
- 저장 항목은 대상 링크 ID, 발송 단계, 발송 경로, 관리 메모, 발송 시각입니다.

## 발송 이력

발송 완료를 기록하면 하객별 누적 발송 횟수와 최초·최근 발송 시각이 갱신됩니다. 리마인드 이력에서는 단계, 하객, 발송 경로, 메모, 발송 시각을 확인할 수 있으며 최대 최근 500건을 화면에 표시합니다.
