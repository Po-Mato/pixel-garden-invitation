# 하객별 초대 링크·QR 발송 센터

관리자 화면 `?admin=invites`에서 하객별 개인 초대 링크를 만들고 열람·RSVP 상태를 확인한다.

## 저장 정보

- 하객 이름
- 신부측 또는 신랑측
- 관계 그룹
- 활성 상태
- 열람 횟수와 최초·최근 열람 시각
- RSVP 완료 시각과 연결된 RSVP 식별자
- 발송 경로, 최초·최근 발송 시각, 발송 횟수, 관리자 메모

전화번호, 계좌 정보, 실제 메시지 본문은 초대 링크에 저장하지 않는다. URL에는 32바이트 무작위 토큰만 들어가며 D1에는 토큰 원문 대신 SHA-256 해시만 저장한다.

## 링크 생성과 보관

1. 관리자 비밀번호로 로그인한다.
2. 한 명 입력 또는 여러 명 입력을 선택한다.
3. 여러 명 입력은 한 줄에 `이름, 신부측 또는 신랑측, 그룹` 순서로 작성한다.
4. 생성 직후 링크를 복사·공유하거나 QR PNG를 내려받는다.

## 명단과 발송 이력

- CSV는 `이름, 측, 그룹` 열을 읽으며 한 번에 최대 100명까지 가져온다.
- CSV 가져오기 후 화면에서 내용을 검토해야 링크 생성 요청을 보낼 수 있다.
- CSV 내보내기는 엑셀 호환 UTF-8 형식이며 발송·열람·RSVP 현황을 포함한다.
- 링크 원문은 현재 관리자 탭에서 확인 가능한 항목만 내보내기에 포함된다.
- 정중한 초대, 따뜻한 초대, 간단한 안내 문구를 선택해 개인별 문구를 복사하거나 공유한다.
- 발송 완료 기록은 카카오톡, 문자, 직접 전달, 기타 중 하나와 선택 메모를 저장한다.
- 같은 하객을 다시 기록하면 발송 횟수가 증가하고 최초 발송 시각은 유지된다.

서버는 원문 토큰을 복구할 수 없다. 생성·재발급된 토큰은 현재 관리자 브라우저 탭의 `sessionStorage`에만 임시 보관된다. 새 탭이나 새 기기에서 링크 원문이 필요하면 해당 링크를 재발급해야 하며 기존 링크는 즉시 폐기된다.

## 하객 경험

- 개인 링크가 유효하면 주소창에서 토큰을 제거하고 브라우저에 안전하게 보관한다.
- 입장 화면에 하객 이름과 관계 그룹을 포함한 맞춤 인사를 표시한다.
- 정원 닉네임과 RSVP 이름·신부/신랑 측을 자동 입력한다.
- 비활성·삭제·잘못된 링크는 존재 여부를 구분해 알리지 않고 일반 초대장으로 전환한다.
- RSVP 저장이 성공한 뒤에만 해당 링크를 응답 완료로 표시한다.

## 개인정보 삭제

초대 링크 데이터는 RSVP 데이터와 같은 `rsvp_delete_at` 정책을 따른다. 현재 일정에서는 예식 30일 후인 2027년 5월 31일에 자동 삭제된다.

## 운영 주의사항

- 링크를 재발급하면 이전 링크와 이전 QR은 사용할 수 없다.
- 하객이 링크를 다른 사람에게 전달할 수 있으므로 개인 링크 자체를 개인정보처럼 취급한다.
- 링크를 잘못 보낸 경우 즉시 중지하거나 재발급한다.
- QR은 외부 QR 서비스가 아니라 브라우저에서 로컬로 생성한다.
