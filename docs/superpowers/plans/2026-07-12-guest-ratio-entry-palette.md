# 하객 비율·진입 화면·색상 개선 구현 계획

1. 하객 6~8번 방향별 원안을 재생성하고 일반/기준선 리뷰를 통과시킨다.
2. 새 원안을 96x144 픽셀 프레임과 배포용 보행 시트로 재현하는 스크립트를 추가한다.
3. 진입 화면 무스크롤과 가로 캐러셀의 CSS 계약 테스트를 추가한다.
4. 진입 화면 레이아웃을 `100dvh` 고정 구조로 바꾸고 캐릭터 목록을 가로 스냅 방식으로 만든다.
5. 청첩장 색상 토큰을 정의하고 주요 UI와 맵 장식의 무지개 파스텔을 절제된 팔레트로 교체한다.
6. 자산 수·크기, 테스트, 타입 검사, 빌드와 모바일 화면을 검증한다.
7. 현재 브랜치를 커밋·푸시하고 GitHub Pages 배포 결과를 확인한다.
