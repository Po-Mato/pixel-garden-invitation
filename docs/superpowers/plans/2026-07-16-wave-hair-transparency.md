# 웨이브 헤어 투명 배경 보정 구현 계획

> **실행 방식:** 현재 세션에서 테스트 우선으로 순차 실행한다. 사용자 요청에 따라 커밋과 푸시는 하지 않는다.

**목표:** 웨이브 헤어 컬 안쪽과 외곽의 흰 매트를 투명화하면서 피부와 밝은 의상을 보존한다.

**구조:** 기존 연결 성분 분석을 공통 모듈로 분리하고 외곽 머리 후보의 크기·밝기 조건을 실제 웨이브 프레임에 맞게 확장한다. 현재 3등신 원본은 유지하고 배포 생성 단계에서 지정된 웨이브 프리셋만 메모리 보정한다.

**기술:** Node.js, Sharp, `node:test`

## 작업

- [x] `scripts/characterAssetGenerator.test.mjs`에 48픽셀보다 큰 외곽 웨이브 틈과 회색 매트가 제거되는 회귀 픽스처를 추가한다.
- [x] 해당 테스트만 실행해 현재 구현에서 실패하는지 확인한다.
- [x] `scripts/lib/guestHairBackground.mjs`로 판정기를 분리하고 피부 경계·중앙 영역 보호 조건을 유지한다.
- [x] `generate-character-assets.mjs`가 세 웨이브 프리셋만 비파괴 보정하도록 연결한다.
- [x] 헤어 회귀 테스트와 기존 피부·흰 의상 보존 테스트를 통과시킨다.
- [x] `pnpm characters:generate`로 배포용 시트를 갱신한다.
- [x] 세 웨이브 프리셋을 마젠타 배경에 합성해 잔여 흰 픽셀이 없는지 확인한다.
- [x] `pnpm characters:audit`와 `pnpm characters:test`를 최종 실행한다.
