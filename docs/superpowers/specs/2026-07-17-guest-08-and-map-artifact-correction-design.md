# 8번 하객 보행 및 맵 아티팩트 보정 설계

## 목표

8번 하객의 좌우 보행이 세 프레임에서 분명하게 읽히도록 원안부터 보정하고, 화장실·동네·예식홀·연회장의 중복 전경과 위치 불일치를 제거한다. 기존 3등신 비율, 맵 동선, 충돌 영역, 포털 이동 계약은 유지한다.

## 8번 하객

- 대상은 `guest-08`의 `left`, `right` 두 방향이다.
- 1컷은 이동 방향 앞발이 전진하고, 2컷은 두 다리가 가까운 중립 자세이며, 3컷은 반대발이 전진한다.
- 세 컷의 머리 크기, 얼굴 형태, 머리카락 부피, 어깨 너비, 드레스 길이, 핸드백 위치와 전체 키를 동일하게 유지한다.
- 기존 좌우 원안을 참고 이미지로 사용하되, 세 컷을 각각 새 보행 자세로 제작한다.
- 승인된 원안을 다시 픽셀화하고 앱용 3열 4방향 보행 시트와 월드용 축소 시트를 갱신한다.
- 정면과 뒷면은 변경하지 않는다.

## 맵 보정

### 화장실

- 오른쪽 변기 칸막이 위에 중복 합성되는 `stall-front` 전경을 런타임과 맵 매니페스트에서 제거한다.
- 배경에 포함된 칸막이와 변기는 유지한다.
- 기존 변기 구역 충돌 영역은 유지한다.
- 전경이 필요하지 않은 맵도 유효하도록 맵 감사 규칙을 조정한다.

### 동네 나무

- 세 수관의 가시 하단과 배경 화단의 그루터기 중심을 기준으로 앵커를 다시 계산한다.
- 수관 이미지와 크기는 유지하고, 필요한 `x`, `y`, `depthY`만 최소 변경한다.
- 세 나무가 같은 높이의 화단에서 서로 다른 높이로 내려앉아 보이지 않도록 기준선을 통일한다.

### 예식홀 버진로드

- 배경 카펫 위에 남아 있는 네 개의 작은 받침 아티팩트를 제거해 카펫 패턴을 복원한다.
- 꽃장식 네 개는 통행 카펫 안쪽이 아니라 좌우 경계 바깥쪽에 번갈아 배치한다.
- 꽃장식은 캐릭터가 앞뒤로 지나갈 때만 자연스럽게 가리는 전경 역할을 유지한다.
- 예식홀 통행 경로와 좌석 영역은 변경하지 않는다.

### 연회장 테이블

- 배경 테이블과 형태가 다른 기존 `table-front`를 배경의 아이보리 천·골드 의자·원근에 맞는 전경으로 교체한다.
- 여섯 테이블의 상단 배경과 전경 접합선이 보이지 않도록 위치를 개별 검증한다.
- 테이블 충돌 영역과 깊이 순서는 유지한다.

## 자산 처리

- 캐릭터와 맵 원본 수정에는 내장 이미지 생성·편집 도구를 사용한다.
- 생성 결과는 캐릭터 시트 또는 투명 전경 이미지인지 눈으로 확인한 뒤에만 저장한다.
- 앱 출력은 기존 Sharp 기반 빌드와 픽셀화 스크립트를 사용해 정해진 크기로 생성한다.
- 변경 대상이 아닌 캐릭터 방향과 맵은 재생성하지 않는다.

## 검증

1. 8번 좌우 원안 시트에서 전진·중립·반대발 전진 실루엣을 확인한다.
2. 8번 픽셀 리뷰와 앱용 보행 시트에서 세 프레임 차이를 확인한다.
3. 전체 전경 감사 시트에서 화장실 중복문 제거, 나무 기준선, 버진로드 가장자리 배치, 테이블 접합을 확인한다.
4. 데스크톱과 모바일에서 네 맵을 직접 캡처해 전경 깊이와 이동 동선을 확인한다.
5. 캐릭터 감사, 맵 감사, 클라이언트 테스트, 타입 검사와 프로덕션 빌드를 통과시킨다.

## 제외 범위

- 8번의 정면·뒷면과 다른 하객 캐릭터는 변경하지 않는다.
- 맵 배경 전체 재생성, 포털·스폰·미니맵·이동 속도 변경은 하지 않는다.
- 사용자 요청 전에는 커밋, 푸시, 배포하지 않는다.
