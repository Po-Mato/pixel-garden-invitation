# 새벽 프리즘 캐릭터 선택 화면 설계

## 목표

- 첫 화면을 현재 정원 맵과 연결되는 화사하고 몽환적인 새벽 프리즘 웨딩 정원으로 재구성한다.
- 선택 캐릭터 이름과 모든 안내 문구가 장식 또는 배경 뒤로 들어가는 문제를 제거한다.
- 캐릭터 12종을 한 화면 안에서 빠르게 비교하고, 선택하고, 닉네임을 입력해 입장하는 흐름을 유지한다.
- `320px` 너비의 작은 모바일 화면부터 데스크톱 폰 프레임까지 세로 스크롤 없이 사용할 수 있어야 한다.

## 시각 방향

화면은 해가 막 떠오르기 전의 유리 온실 정원을 표현한다. 배경에는 짙은 녹색 대신 새벽 청록, 연보라, 코랄, 진주빛 아이보리와 무광 골드를 함께 사용한다. 한 가지 색 계열이 화면을 지배하지 않도록 상단 하늘, 중앙 정원, 하단 조작 영역의 색 역할을 구분한다.

- 상단은 옅은 청록 하늘과 작은 별빛, 프리즘 조각, 흩날리는 꽃잎으로 구성한다.
- 중앙은 꽃 아치, 양쪽 화단, 바닥 모자이크가 있는 캐릭터 무대로 만든다.
- 하단 선택 영역은 반투명 유리 느낌이 아니라 충분히 불투명한 종이·진주 패널로 처리해 글자 대비를 보장한다.
- 픽셀 감성은 `1px`~`4px` 선, 계단식 모서리, `steps()` 애니메이션으로 유지한다.
- 큰 흐림 효과, 장식용 그라디언트 구체, 과도하게 둥근 카드, 외부 이미지 자산은 사용하지 않는다.

## 화면 구조

### 초대장 헤더

- 작은 영문 문구, 커플 이름, 한 줄 안내를 독립된 전면 레이어로 배치한다.
- 제목 뒤에는 얇은 금빛 장식과 작은 별빛만 두고, 글자 영역에는 복잡한 무늬를 두지 않는다.
- 안내 문구는 배경색과 충분한 명도 차이를 유지하고 작은 화면에서도 한 줄 또는 자연스러운 두 줄로만 표시한다.

### 캐릭터 무대

- 선택 캐릭터는 화면의 가장 강한 초점이며 기존 96x144 미리보기 규격을 유지한다.
- 캐릭터 뒤에는 원형 광륜 대신 유리 온실 창, 꽃 아치, 바닥 프리즘 모자이크를 계층적으로 배치한다.
- 선택 캐릭터 이름은 무대 하단의 불투명한 리본 명패에 표시한다.
- 캐릭터, 이름 명패, 조작 버튼은 모든 장식 의사 요소보다 높은 쌓임 순서를 갖는다.
- 미세한 별빛과 꽃잎 애니메이션은 캐릭터를 가리지 않는 가장자리에서만 작동한다.

### 빠른 선택과 캐릭터 목록

- 무작위 선택과 기본 캐릭터는 아이콘과 짧은 문구가 있는 보조 버튼으로 유지한다.
- 12개 캐릭터는 기존 가로 스냅 캐러셀을 유지하되 썸네일 비중을 키우고 라벨 영역을 별도로 확보한다.
- 카드 라벨에는 `word-break: keep-all`과 제한된 두 줄 높이를 사용해 한글이 한 글자씩 끊기지 않게 한다.
- 선택 카드는 금빛 프레임, 코랄 강조선, 체크 표시와 약한 상승 효과로 명확히 구분한다.
- 캐러셀 양쪽에는 다음 카드가 일부 보이게 해 가로 탐색 가능성을 자연스럽게 드러낸다.

### 닉네임과 입장

- 닉네임 입력과 입장 버튼은 화면 최하단의 고정된 조작 밴드로 묶는다.
- 입력값과 라벨은 항상 불투명한 배경 위에 표시한다.
- 입장 버튼은 활성 상태에서 코랄·금빛·진주 하이라이트를 사용하고, 비활성 상태도 글자를 읽을 수 있는 대비를 유지한다.
- 기존 제출, 공백 제거, 저장 동작은 변경하지 않는다.

## 레이어 계약

선택 화면 내부는 다음 순서를 따른다.

1. 배경 하늘과 바닥 패턴
2. 온실 창, 꽃 아치, 화단, 별빛과 꽃잎
3. 캐릭터 광원과 바닥 그림자
4. 캐릭터 스프라이트
5. 캐릭터 이름 명패, 헤더 문구, 카드 라벨
6. 버튼, 입력, 포커스 링

장식 의사 요소는 `pointer-events: none`을 사용한다. 텍스트 컨테이너에는 명시적인 `position`과 `z-index`를 부여해 새 쌓임 맥락에서도 장식보다 항상 앞에 있도록 한다.

## 반응형 동작

- 기본 폰 프레임은 기존 `430px` 최대 너비와 `100dvh` 고정 구조를 유지한다.
- 높이가 짧은 화면에서는 장식 크기와 여백을 줄이고 캐릭터나 텍스트 자체를 찌그러뜨리지 않는다.
- `320x568`에서는 헤더, 미리보기, 캐러셀, 입력, 입장 버튼이 모두 화면 안에 남아야 한다.
- `390x844`와 `430x932`에서는 중앙 무대에 추가 높이를 배분해 캐릭터를 더 풍성하게 보여준다.
- 긴 카드 이름은 최대 두 줄이며 버튼 바깥으로 넘치거나 이전·다음 카드와 겹치지 않아야 한다.

## 접근성과 모션

- 현재 `aria-label`, `aria-pressed`, 실제 버튼 구조를 유지한다.
- 키보드 포커스는 밝은 아이보리 외곽선과 짙은 잉크색 보조선으로 표시한다.
- 장식은 클릭과 가로 스크롤을 방해하지 않는다.
- `prefers-reduced-motion`에서는 별빛, 꽃잎, 선택 카드의 부유 애니메이션을 정지한다.
- 텍스트 대비와 버튼 상태는 색상 하나에만 의존하지 않는다.

## 구현 범위

- `EntryScreen`에는 헤더 장식과 하단 조작 밴드를 위한 의미 없는 장식 요소 또는 구조 클래스만 최소한으로 추가한다.
- `CharacterCustomizer`에는 무대 장식, 선택 명패, 카드 라벨용 구조 클래스를 추가한다.
- 주된 변화는 `styles.css`의 진입 화면 관련 규칙에 한정한다.
- 캐릭터 자산, 캐릭터 상태 모델, 저장 키, 게임 입장 데이터 흐름은 변경하지 않는다.

## 검증 기준

- 선택 캐릭터 이름, 헤더 안내, 카드 이름, 닉네임 라벨과 값, 입장 버튼 문구가 모든 장식보다 앞에 표시되어야 한다.
- 긴 한국어 캐릭터 이름이 한 글자씩 세로로 분리되지 않고 최대 두 줄로 읽혀야 한다.
- 12개 캐릭터가 가로 스냅 캐러셀에서 선택 가능해야 한다.
- `320x568`, `390x844`, `430x932`에서 세로 스크롤, 잘림, 텍스트·캐릭터·버튼 겹침이 없어야 한다.
- 선택 상태, 무작위 선택, 기본 캐릭터, 닉네임 입력, 입장 동작이 기존과 동일하게 작동해야 한다.
- 일반 모션과 모션 감소 설정 모두에서 레이아웃이 유지되어야 한다.
- 브라우저 스크린샷과 DOM 위치 검사를 수행한다.
- `pnpm test`, `pnpm typecheck`, 프로덕션 `pnpm build`가 모두 통과해야 한다.
