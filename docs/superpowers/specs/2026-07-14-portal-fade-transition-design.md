# 포털 페이드 전환 설계

## 목표

모든 포털 이동에서 캐릭터가 포털 접근 타일에 도착한 모습을 먼저 보여주고, 짧은 정지와 페이드아웃·페이드인을 거쳐 자연스럽게 다음 맵으로 전환한다.

## 전환 흐름

1. 조이스틱 이동 또는 포털 클릭 자동 보행으로 포털의 `approach` 타일에 도착한다.
2. 기존 맵의 접근 타일에 캐릭터를 배치하고 포털의 `facing` 방향으로 세운다.
3. 이동을 정지하고 `0.15초` 동안 도착 모습을 유지한다.
4. `0.25초` 동안 화면을 페이드아웃한다.
5. 화면이 완전히 가려진 시점에 포털의 `to` 맵과 `spawn` 위치로 변경한다.
6. 새 맵을 `0.3초` 동안 페이드인한다.
7. 전환을 종료하고 이동 입력을 다시 허용한다.

전체 전환 시간은 약 `0.7초`다.

## 상태와 책임

`GameWorld`는 다음 전환 단계를 명시적으로 관리한다.

- `idle`: 일반 플레이 상태
- `arrival`: 포털 접근 타일 도착 후 정지 상태
- `fade-out`: 기존 맵을 가리는 상태
- `fade-in`: 새 맵을 드러내는 상태

조이스틱 진입과 포털 클릭 자동 보행은 하나의 포털 전환 시작 함수를 사용한다. 이 함수는 도착 좌표와 방향을 먼저 반영하고, 이동·목표·조이스틱 상태를 초기화한 뒤 전환 상태를 시작한다.

도착과 페이드인 단계는 정리 가능한 타이머로 처리한다. 일반 모션의 맵 변경은 opacity `transitionend`에서 실행해 실제 완전 불투명 시점을 보장하고, event 유실에 대비한 `1000ms` fallback만 둔다. 모션 감소 환경에서는 CSS transition이 없으므로 `250ms` 상태 타이머로 맵을 변경한다. phase ref로 중복 event와 오래된 fallback 실행을 무시한다.

## 화면과 입력

`GameWorld` 최상단에 전체 월드를 덮는 단색 전환 레이어를 둔다. 레이어는 평상시 투명하고 포인터 입력을 받지 않으며, 전환 중에는 불투명도와 포인터 차단을 단계에 맞게 적용한다.

전환 중에는 다음 입력을 모두 무시한다.

- 조이스틱과 방향키 이동
- 맵 클릭 이동
- 포털 클릭
- 초대장 메뉴, 장소 상세, NPC 선택

전환 시작 시 조이스틱 벡터와 타일 반복 입력을 초기화한다. 이때 실제로 누르고 있던 포인터 ID와 방향키 목록은 release latch가 추적한다. 방향키 `keyup`은 window에서도 받아 포커스가 이동한 뒤의 해제를 놓치지 않는다. 관계없는 다른 키의 `keyup`으로 잠금이 풀리지 않으며, 원래 입력 자체가 해제된 뒤에만 새 이동을 허용한다. 클릭 자동 보행처럼 누른 조이스틱 입력이 없던 전환은 페이드인 종료 직후 바로 이동할 수 있다.

`prefers-reduced-motion: reduce` 환경에서는 전환 상태와 입력 잠금은 유지하되 불투명도 애니메이션을 제거한다.

## 실시간 이동

접근 타일에 도착할 때 client throttle을 우회해 기존 맵의 최종 정지 위치와 포털 facing을 전송한다. Worker는 같은 zone·좌표, 증가한 seq, `moving=false`이며 이동 종료 또는 방향 변경인 상태 변경만 `100ms` 제한 창당 한 번 수락한다. 같은 방향의 중복 정지, 방향 교대 정지 spam, 일반 이동 spam은 계속 제한한다. 맵이 실제로 바뀌는 시점에는 기존 `moveToZone`을 사용해 새 맵의 `spawn`과 정지 상태를 전송한다. 목적 맵 정보가 페이드아웃 이전에 전송되지 않도록 한다.

## 검증

- 조이스틱으로 접근 타일에 도착한 직후 기존 맵과 도착 좌표가 유지되는지 확인한다.
- `0.15초` 정지와 `0.25초` 페이드아웃 중에는 맵이 바뀌지 않는지 확인한다.
- opacity `transitionend` 시 불투명도 `1`에서 목적 맵의 `spawn`으로 변경되는지 확인한다.
- `0.3초` 페이드인 동안 입력이 무시되고 완료 후 다시 이동할 수 있는지 확인한다.
- 포털 클릭 자동 보행에도 같은 전환 흐름이 적용되는지 확인한다.
- held input은 원래 키·포인터 해제 전까지 유지되고, 관계없는 release로 풀리지 않는지 확인한다.
- 모션 감소 환경에서 computed transition이 `none`이고 상태 타이밍은 유지되는지 확인한다.
- 기존 맵 클릭, 미니맵, 메뉴, 실시간 이동 및 Worker 제한 테스트가 계속 통과하는지 확인한다.
