# 연회장 재생성과 화장실 동선 재배치 설계

## 목표

현재 예식홀 끝에 연결된 연회장을 로비 오른쪽의 기존 화장실 동선 위치로 옮기고, 연회장에서 화장실로 이어지는 순차 동선을 만든다. 접합이 깨진 반원 테이블 합성 자산은 폐기하고, 연회장 배경과 완전한 원형 테이블 전경을 처음부터 새로 생성한다.

## 확정 범위

- 전체 여정 순서를 `예식홀 → 연회장 → 화장실`로 바꾼다.
- 로비 오른쪽 포털은 화장실 대신 연회장으로 연결한다.
- 예식홀과 연회장의 직접 연결을 제거한다.
- 연회장은 로비와 화장실 사이의 순차 구역이 된다.
- 연회장 크기는 기존 `1200x930`을 유지한다.
- 원형 테이블은 4개만 배치하고 중앙 이동 통로를 넓게 확보한다.
- 연회장 배경과 테이블 전경 2종을 기존 자산을 수정하지 않고 새로 생성한다.
- 기존 `table-front` 상하 분할 합성 방식은 제거한다.
- 이동, 미니맵, 포털 타일, 캐릭터 깊이, 전체 검증과 GitHub Pages 배포를 포함한다.

## 월드 그래프

변경 후 관련 구역의 양방향 연결은 다음과 같다.

```text
예식홀 ↔ 로비 ↔ 연회장 ↔ 화장실
             ↕
         신부 대기실
```

세부 계약은 다음과 같다.

- 로비의 기존 `lobby-to-restroom` 포털을 연회장 포털로 교체한다.
- 로비 오른쪽의 포털 위치와 접근 방향은 유지하되 라벨, 목적지와 도착 스폰만 연회장 기준으로 바꾼다.
- 예식홀의 `hall-to-banquet` 포털을 제거한다.
- 예식홀에는 로비 복귀 포털만 남긴다.
- 연회장 왼쪽에는 로비 복귀 포털을 둔다.
- 연회장 오른쪽에는 화장실 진입 포털을 둔다.
- 화장실 왼쪽 포털은 로비가 아니라 연회장으로 돌아간다.
- 포털 클릭 시 가장 가까운 진입 타일까지 걸어간 뒤 전환하는 기존 동작과 조이스틱 진입 동작을 유지한다.
- 포털은 방향별 3타일 진입 영역과 현재 fade 전환을 유지한다.

## 여정 순서와 문구

구역 배열, 공유 프로토콜, 자산 매니페스트와 미니맵 여정 순서를 다음과 같이 일치시킨다.

```text
home
neighborhood
subway-station
subway-train
venue-exterior
lobby
bridal-room
ceremony-hall
banquet
restroom
```

- `ceremony-hall`의 `journeyIndex`는 `7`을 유지한다.
- `banquet`의 `journeyIndex`는 `8`로 바꾼다.
- `restroom`의 `journeyIndex`는 `9`로 바꾼다.
- 연회장 설명에서 여정이 끝난다는 표현을 제거하고 식사와 축하 메시지에 집중한다.
- 화장실 설명은 연회장 옆에서 잠시 정돈하는 공간임을 나타낸다.

## 새 연회장 공간 구성

연회장은 `1200x930` 가로형 공간으로 유지하고 왼쪽에서 오른쪽으로 이동하는 구조로 다시 설계한다.

- 왼쪽 중앙: 로비 출입구와 3타일 세로 포털
- 오른쪽 중앙: 화장실 출입구와 3타일 세로 포털
- 위쪽 벽: 큰 창, 자연광, 긴 뷔페와 음료 코너
- 중앙: 좌우 포털을 잇는 넓은 가로 통로
- 위쪽 좌우: 원형 테이블 2개
- 아래쪽 좌우: 원형 테이블 2개
- 아래쪽 우측 벽 주변: 축하 메시지 인터랙션과 장식 콘솔
- 벽과 코너: 꽃 장식, 간접 조명, 가랜드

중앙 통로와 각 테이블 둘레에는 최소 한 타일 이상의 연속 보행 공간을 둔다. 포털 접근점, 축하 메시지 인터랙션과 테이블 충돌 영역은 서로 겹치지 않는다.

## 이미지 자산 구조

### 배경

`map-assets/reference/v2/banquet/pixel-background-source.png`을 새 이미지로 교체한다.

- 고밀도 모바일 게임 픽셀 아트
- 정사영 탑다운 3/4 시점
- 밝고 화사한 샴페인, 민트, 코랄 포인트
- 창, 자연광, 바닥, 벽, 뷔페, 음료 코너, 꽃과 조명 포함
- 왼쪽과 오른쪽 출입구를 같은 높이로 정렬
- 테이블, 의자, 인물, 글자, 로고와 UI는 포함하지 않음
- 최종 빌드 크기 `1200x930`

### 완전한 테이블 전경

다음 두 자산을 새로 생성한다.

```text
map-assets/reference/v2/banquet/table-floral-source.png
map-assets/reference/v2/banquet/table-dining-source.png
```

- `table-floral`: 꽃과 캔들 중심의 아이보리 상차림
- `table-dining`: 음식, 식기와 유리잔 중심의 샴페인 상차림
- 각 자산은 테이블 상판, 테이블보, 모든 의자와 바닥 접촉부를 포함한 완전한 한 개의 원형 테이블이다.
- 테이블을 상단과 하단으로 나누지 않는다.
- 내장 `image_gen`에서 평면 마젠타 크로마 배경으로 생성한 뒤 로컬 helper로 투명 PNG를 만든다.
- 투명 모서리, 유효한 알파, 테두리 잔여색과 잘린 의자가 없는지 검사한다.
- 최종 전경은 동일한 고정 크기로 빌드하고 두 종류를 대각선으로 각각 2회 배치한다.

기존 아래 자산과 참조는 소비 코드에서 제거하고 파일도 삭제한다.

```text
map-assets/reference/v2/banquet/table-front-source.png
client/public/assets/maps/v2/banquet/table-front.png
```

## 렌더링과 깊이

- 배경에는 테이블을 그리지 않는다.
- 각 테이블은 완전한 투명 전경 하나로 렌더링한다.
- 테이블별 `depthY`는 테이블 하단의 바닥 접촉선에 맞춘다.
- 캐릭터의 깊이가 테이블보다 작으면 테이블 뒤에, 크면 테이블 앞에 표시된다.
- 4개 테이블은 서로 겹치지 않고 중앙 통로를 침범하지 않는다.
- 뷔페와 벽 장식은 배경에 포함해 별도 접합 전경을 만들지 않는다.
- 기존 `banquet-light`의 느린 밝기 효과는 새 배경과 충돌하지 않는 범위에서 유지한다.

## 이미지 생성 품질 게이트

각 이미지는 한 자산씩 생성하고 다음 조건을 통과한 결과만 저장한다.

- 연회장 배경에 인물, 글자, 로고 또는 UI가 없어야 한다.
- 시점이 다른 가구, 벽을 뚫는 장식, 휘어진 창과 잘린 뷔페가 없어야 한다.
- 왼쪽과 오른쪽 출입구가 같은 이동 축에 있어야 한다.
- 테이블 소스에는 정확히 한 개의 완전한 원형 테이블만 있어야 한다.
- 의자, 테이블보, 식기와 꽃이 프레임 밖에서 잘리지 않아야 한다.
- 크로마 배경은 단색이어야 하고 피사체에 같은 마젠타 색을 사용하지 않는다.
- 결과가 조건을 어기면 기존 이미지와 합성하거나 부분 보정하지 않고 해당 자산만 다시 생성한다.

## 데이터와 자산 계약

- 자산 매니페스트의 구역 순서를 `ceremony-hall`, `banquet`, `restroom` 순으로 바꾼다.
- 연회장 오버레이를 `table-floral.png`, `table-dining.png` 두 항목으로 바꾼다.
- `requiredArtifacts`는 창, 뷔페, 중앙 통로, 원형 테이블, 상차림, 꽃과 출입구를 반영한다.
- 전경 감사 렌더러는 새 4개 테이블 배치와 새 파일명을 사용한다.
- 월드와 realtime 경계는 연회장 `1200x930`, 화장실 `660x660`을 유지한다.
- 연회장 기본 스폰은 왼쪽 로비 출입구 안쪽에 둔다.
- 화장실에서 돌아오는 연회장 스폰은 오른쪽 출입구 안쪽에 둔다.
- 화장실 기본 스폰은 기존 왼쪽 출입구 안쪽을 유지한다.

## 테스트

### 자동 검증

- 정확한 10개 구역 순서와 `journeyIndex`를 검증한다.
- 포털 그래프가 `예식홀 ↔ 로비 ↔ 연회장 ↔ 화장실`인지 검증한다.
- `hall-to-banquet`, `lobby-to-restroom`, `banquet-to-hall`, `restroom-to-lobby`가 남지 않았는지 검증한다.
- 새 포털의 접근점, 도착 스폰과 세 진입 타일이 보행 가능하고 비충돌인지 검증한다.
- 로비에서 연회장, 연회장에서 화장실까지 경로 탐색이 성공하는지 검증한다.
- 연회장 4개 테이블의 충돌 영역과 중앙 통로가 겹치지 않는지 검증한다.
- 두 테이블 출력이 지정 크기, 투명 알파와 유효 픽셀을 갖는지 자산 감사로 검증한다.
- `table-front`가 매니페스트, 월드 장식과 렌더링 테스트에서 제거됐는지 검증한다.
- 축하 메시지 인터랙션이 연회장에 유지되는지 검증한다.
- 미니맵, 포털 클릭, 조이스틱 진입, fade 전환과 UI 클릭 차단 회귀 테스트를 실행한다.

### 시각 검증

데스크톱 `1440x900`, 모바일 `390x844`, 가로 `844x390`에서 다음을 확인한다.

1. 로비 오른쪽 포털이 연회장으로 표시되고 배경 출입구 중앙과 일치한다.
2. 연회장 좌우 포털이 출입구와 같은 위치에 있다.
3. 원형 테이블 4개가 완전하며 접합선, 잘린 의자와 중복 상판이 없다.
4. 두 종류의 테이블이 대각선으로 배치되고 반복감이 과하지 않다.
5. 캐릭터가 각 테이블의 앞과 뒤에서 자연스럽게 가려진다.
6. 중앙 통로를 조이스틱과 포털 클릭 이동으로 막힘 없이 통과한다.
7. 화장실 복귀 포털이 연회장으로 연결된다.
8. 미니맵, 라벨, 조이스틱과 초대장 메뉴가 맵 이동 입력을 발생시키지 않는다.

## 배포

- 맵 자산 감사, 캐릭터 자산 감사, 전체 테스트, 타입 검사와 프로덕션 빌드를 통과한다.
- 관련 설계, 월드 데이터, 이미지 자산, 매니페스트와 테스트만 명시적으로 스테이징한다.
- 기존 미추적 캐릭터 원안 디렉터리는 변경하거나 스테이징하지 않는다.
- 현재 `main` 워크트리에서 커밋하고 `origin/main`에 푸시한다.
- GitHub Pages 워크플로가 성공할 때까지 기다린다.
- 공개 URL에서 새 연회장 자산, 포털 그래프와 라이브 화면을 다시 확인한다.
