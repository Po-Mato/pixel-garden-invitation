# 다중 타일 포털과 조이스틱 손잡이 가시성 설계

## 목표

- 포털의 시각적 위치와 실제 맵 전환 판정을 `30x30px` 이동 타일에 정확히 맞춘다.
- 조이스틱으로 포털에 진입할 때 한 칸을 정밀하게 맞춰야 하는 불편을 줄인다.
- 포털 빛이 하객 캐릭터를 덮지 않도록 깊이 순서를 바로잡는다.
- 기존 웨딩 나침반 조이스틱의 디자인을 유지하면서 손잡이를 조금 더 선명하게 보이게 한다.
- 기존 타일 보행, 포털 도착 지연, fade 전환, 미니맵과 접근성 동작은 유지한다.

## 포털 데이터 구조

`WorldPortal`에 실제 진입 가능한 세 타일의 중심 좌표를 담는 `entryTiles: Point[]`를 둔다. 기존 `approach`는 세 타일의 중앙 좌표로 유지하되, 포털 판정과 자동 보행 목적지는 `entryTiles`를 기준으로 한다.

- `facing`이 `up` 또는 `down`이면 중앙 타일의 좌우 `30px` 위치를 더해 가로 3타일을 만든다.
- `facing`이 `left` 또는 `right`이면 중앙 타일의 상하 `30px` 위치를 더해 세로 3타일을 만든다.
- 모든 타일 중심은 월드의 `30px` 이동 격자에 정렬한다.
- 기존 원형 포털용 `visualCenter`와 `80x34px` 타원 판정은 제거한다.
- `portalEntryRect`는 세 타일을 감싸는 `90x30px` 또는 `30x90px` 사각형을 반환한다.
- `pointInPortalEntry`는 세 진입 타일 중 하나에 포함되는지를 판정한다.

포털 생성 헬퍼가 `approach`와 `facing`으로 `entryTiles`를 구성하므로 18개 포털 정의에 중복 좌표를 수동으로 나열하지 않는다. 테스트에서 모든 생성 결과가 월드 경계 안의 이동 가능한 타일인지 검증한다.

## 이동과 전환

### 조이스틱 이동

캐릭터가 세 진입 타일 중 어느 칸의 중심에 도착해도 기존 도착 지연과 fade 전환을 시작한다. 인접한 포털 바깥 타일에서는 전환하지 않는다. 조이스틱을 누른 상태에서 전환할 때 입력 해제 래치와 새 맵 연속 이동 방지 동작은 그대로 유지한다.

### 포털 클릭

포털 띠를 클릭하면 현재 위치에서 각 진입 타일까지 경로를 계산한다. 도달 가능한 경로 중 타일 수가 가장 짧은 경로를 선택해 자동 보행한다. 경로 길이가 같으면 중앙 타일을 우선해 동작을 결정적으로 유지한다.

세 타일 모두 도달할 수 없으면 자동 보행을 시작하지 않고 기존 경로 실패 안내를 표시한다. 이동 중 조이스틱 입력, 맵 클릭 또는 UI 조작에 따른 취소 규칙은 변경하지 않는다.

## 포털 시각 구성

- 포털 버튼의 월드 좌표 크기를 실제 판정 범위와 같은 `90x30px` 또는 `30x90px`로 렌더링한다.
- 버튼 내부에 세 개의 `30x30px` 사각 발광 타일을 빈틈없이 배치한다.
- 각 타일은 청백색 외곽선, 내부 격자와 중심 문양으로 이동 가능한 칸을 명확히 보여준다.
- 수직 빛기둥과 상승 입자는 세 번 중복하지 않고 포털 띠 전체에 하나만 표시한다.
- 목적지 라벨과 선택 시 금빛 강조는 유지한다.
- `prefers-reduced-motion`에서는 빛기둥, 입자와 타일 맥동을 정지한 정적 발광 상태로 표시한다.

포털은 `worldDepth(approach.y) - 100` 깊이에 렌더링한다. 빛기둥이 차지하는 높이 안에서 플레이어, 원격 하객과 NPC의 깊이가 포털보다 높아지므로 캐릭터가 포털 앞에 자연스럽게 표시된다. 정보 버튼은 기존 `z-index: 9000`을 유지한다.

## 미니맵

미니맵은 `portalEntryRect`가 반환하는 실제 3타일 범위를 표시한다. 목표 포털 강조도 같은 범위에 적용한다. 미니맵 클릭이 캐릭터 이동으로 전달되지 않는 기존 이벤트 차단은 유지한다.

## 조이스틱 J2 스타일

기존 이미지 에셋과 입력 로직은 변경하지 않고 CSS 표시만 조정한다.

- 데스크톱 손잡이: `34px`에서 `38px`로 확대
- 데스크톱 이동 반경: `30px`에서 `26px`로 조정
- 모바일 손잡이: `27px`에서 `30px`로 확대
- 모바일 이동 반경: `22px`에서 `21px`로 조정
- 손잡이: 채도와 대비를 소폭 높이고 얇은 장미색 아래 윤곽과 밝은 하이라이트 그림자를 추가
- 비활성 상태와 키보드 포커스 상태: 기존 의미와 접근성 유지

이 크기와 이동 반경 조합은 손잡이가 최대로 이동해도 `90x90px` 데스크톱 베이스와 `72x72px` 모바일 베이스를 벗어나지 않는다.

## 테스트와 검증

### 자동 테스트

- 18개 포털이 각각 정확히 3개, 전체 54개의 진입 타일을 생성하는지 검사한다.
- 상·하 포털이 가로 3칸, 좌·우 포털이 세로 3칸인지 검사한다.
- 모든 진입 타일이 `30px` 격자, 월드 경계와 이동 가능 영역 계약을 만족하는지 검사한다.
- 각 진입 타일 내부는 포털로 판정되고 바로 인접한 바깥 지점은 판정되지 않는지 검사한다.
- 포털 클릭이 도달 가능한 최단 진입 타일을 선택하는지 검사한다.
- 조이스틱으로 중앙과 양 끝 진입 타일에 도착했을 때 모두 전환되고 바깥 인접 타일에서는 전환되지 않는지 검사한다.
- 포털이 캐릭터보다 낮은 깊이에, 정보 버튼은 기존 상단 깊이에 렌더링되는지 검사한다.
- 포털 버튼과 미니맵 표식이 같은 3타일 범위를 사용하는지 검사한다.
- 조이스틱이 데스크톱과 모바일의 새 손잡이 크기, 이동 반경과 대비 스타일을 사용하는지 검사한다.
- 전체 테스트, 타입 검사와 프로덕션 빌드를 실행한다.

### 시각 검증

- 데스크톱과 모바일에서 상·하 포털의 가로 3칸과 좌·우 포털의 세로 3칸이 입구와 보행로 안에 놓이는지 확인한다.
- 세 칸의 경계가 분명하고 하나의 포털 띠와 하나의 수직 빛기둥으로 읽히는지 확인한다.
- 캐릭터가 포털 타일과 빛기둥보다 앞에 표시되는지 확인한다.
- 조이스틱 손잡이가 중립과 상·하·좌·우 끝 위치에서 잘 보이고 베이스를 벗어나지 않는지 확인한다.
- 미니맵, 조이스틱과 초대장 메뉴 조작이 맵 클릭 이동으로 전달되지 않는지 확인한다.

## 배포

검증을 통과한 변경을 `main`에 커밋하고 푸시해 GitHub Pages 배포를 실행한다. 워크플로 완료 후 공개 URL에서 새 번들 로드, 포털 3타일 표시, 캐릭터 깊이 관계와 조이스틱 손잡이 스타일을 다시 확인한다.
