# 예식 일정·장소 및 캘린더 저장 설계

## 목표

하객이 게임에 입장하기 전과 입장한 후 모두 예식 일시와 장소를 빠르게 확인하고, 사용하는 캘린더에 일정을 저장할 수 있게 한다. 입장 화면과 초대장 메뉴는 하나의 공통 예식 데이터를 사용해 정보 불일치를 방지한다.

## 확정 예식 정보

- 신랑: 이승재
- 신부: 이건희
- 일정 제목: `이승재 · 이건희 결혼식`
- 시작: 2027년 5월 1일 오후 5시 10분
- 종료: 2027년 5월 1일 오후 6시 40분
- 시간대: `Asia/Seoul`
- 장소: MJ컨벤션
- 홀: 5층 파티오볼룸
- 주소: 경기 부천시 소사구 경인로 386
- 캘린더 알림: 별도 지정하지 않고 사용자 캘린더의 기본 설정 사용

## 범위

### 포함

- 기존 샘플 신랑·신부, 연도, 예식 일시 및 장소 정보 교체
- 입장 화면에 간결한 예식 요약과 캘린더 저장 진입점 표시
- 초대장 메뉴에 상세 예식 정보, 주소 복사, 캘린더 저장 진입점 표시
- 기본 캘린더용 `.ics` 파일 생성
- Google Calendar 일정 작성 링크 생성
- 전체 일정 문구 복사
- 주소 복사
- 복사 및 외부 열기 실패 상태 안내
- 키보드 포커스, 스크린 리더 알림, 맵 입력 차단
- 단위·컴포넌트·통합 테스트

### 제외

- 네이버 지도, 카카오맵, Google 지도 길 찾기 연동
- 캘린더 알림 시간 강제 지정
- 서버 또는 데이터베이스 저장
- 실시간 이동, 포털, 맵 경로 변경
- RSVP 흐름 변경

## 공통 데이터 모델

`shared/src/content.ts`의 `invitationContent`가 예식 정보의 단일 출처가 된다. 화면 표시용 한국어 문장을 저장하지 않고 구조화된 원본 데이터를 저장한다.

예식 정보는 다음 의미를 표현해야 한다.

- 커플 이름과 역할
- 시작·종료 시각과 IANA 시간대
- 장소명, 층·홀, 전체 주소
- 캘린더 제목과 설명에 사용할 값

시작·종료 시각은 오프셋이 포함된 ISO 8601 값으로 정의한다.

```text
2027-05-01T17:10:00+09:00
2027-05-01T18:40:00+09:00
```

표시 문구, `.ics`, Google Calendar URL, 복사용 문구는 모두 이 구조화된 데이터에서 파생한다. 기존 `spots` 정보는 그대로 유지하되 예식 안내 문구가 공통 데이터와 충돌하지 않도록 함께 갱신한다.

## 구성요소

### WeddingEventSummary

입장 화면과 초대장 메뉴가 함께 사용하는 예식 정보 컴포넌트다.

- `compact`: 입장 화면용. 날짜, 시작 시각, 장소와 홀, 캘린더 저장 버튼을 표시한다.
- `detail`: 초대장 메뉴용. 시작·종료 시각, 장소, 홀, 전체 주소, 주소 복사 버튼, 캘린더 저장 버튼을 표시한다.

두 형태는 같은 데이터 포매터를 사용한다. 화면별로 날짜나 장소 문자열을 다시 작성하지 않는다.

### CalendarSaveSheet

`캘린더 저장` 버튼에서 여는 하단 선택창이다.

- 기본 캘린더: `.ics` 파일을 생성해 연다.
- Google 캘린더: 완성된 일정 작성 URL을 일반 외부 링크로 연다.
- 일정 복사: 제목, 날짜, 시작·종료 시각, 장소, 홀, 주소를 하나의 문구로 복사한다.

선택창은 모달 대화상자로 동작한다. 열릴 때 첫 동작으로 포커스를 이동하고, 닫을 때 원래 누른 버튼으로 포커스를 복원한다. 배경과 맵으로 포인터 및 키보드 입력이 전달되지 않아야 한다.

### 캘린더 유틸리티

클라이언트 전용 순수 함수로 다음을 담당한다.

- 표시용 날짜·시간 포매팅
- `.ics` 텍스트 생성
- Google Calendar URL 생성
- 주소 및 일정 복사용 문구 생성

UI나 브라우저 전역 객체에 의존하지 않는 생성 함수와 다운로드·복사를 수행하는 브라우저 어댑터를 분리한다.

## 화면 설계

### 입장 화면

헤더의 기존 샘플 이름과 연도를 실제 정보로 교체한다. 헤더 바로 아래, 캐릭터 선택기 위에 `compact` 예식 요약을 배치한다.

표시 순서는 다음과 같다.

1. `이승재 & 이건희의 정원`
2. `2027년 5월 1일 토요일 오후 5시 10분`
3. `MJ컨벤션 5층 파티오볼룸`
4. `캘린더 저장`
5. 기존 캐릭터 선택, 닉네임, 입장 버튼

요약은 별도 중첩 카드로 만들지 않고 입장 헤더와 캐릭터 선택 사이의 정돈된 정보 영역으로 표현한다. 작은 모바일 화면에서도 캐릭터 선택의 첫 행이 지나치게 아래로 밀리지 않아야 한다.

### 초대장 메뉴

기존 메뉴 시트 상단에 `detail` 예식 요약을 배치하고 기존 바로가기 그리드는 아래에 유지한다. 주소는 선택 가능한 텍스트로 표시하며 복사 버튼은 주소만 복사한다.

메뉴 내부의 예식 정보와 캘린더 선택창을 누르는 동작은 캐릭터 이동이나 맵 클릭으로 처리되지 않아야 한다.

## 캘린더 형식

### 기본 캘린더

`.ics`는 UTF-8, CRLF 줄바꿈으로 생성하며 최소한 다음 필드를 포함한다.

- `VCALENDAR`
- `VERSION:2.0`
- `PRODID`
- `VEVENT`
- 안정적인 `UID`
- `DTSTAMP`
- `DTSTART`
- `DTEND`
- `SUMMARY`
- `LOCATION`
- `DESCRIPTION`

날짜는 한국 시간의 실제 순간을 유지하도록 UTC로 변환해 기록한다. 텍스트의 역슬래시, 쉼표, 세미콜론, 줄바꿈은 iCalendar 규칙에 맞게 이스케이프한다. 강제 알림을 위한 `VALARM`은 추가하지 않는다.

파일명은 운영체제에서 안전한 ASCII 이름인 `wedding-event.ics`를 사용한다. 다운로드 후 생성한 객체 URL은 정리한다.

### Google Calendar

Google Calendar 템플릿 URL에 제목, UTC 시작·종료 시각, 장소, 설명을 URL 인코딩해 넣는다. 팝업 차단을 피하기 위해 사용자 클릭에 직접 연결된 일반 외부 링크로 열며 새 탭 사용을 명시한다.

### 일정 복사

복사 문구는 다음 정보를 줄 단위로 포함한다.

```text
이승재 · 이건희 결혼식
2027년 5월 1일 토요일 오후 5시 10분 - 오후 6시 40분
MJ컨벤션 5층 파티오볼룸
경기 부천시 소사구 경인로 386
```

## 상태와 오류 처리

- 복사 성공 시 해당 버튼 주변과 `aria-live` 영역에 `주소를 복사했습니다` 또는 `일정을 복사했습니다`를 표시한다.
- 복사 실패 시 원문을 선택 가능한 상태로 유지하고 `복사하지 못했습니다. 내용을 길게 눌러 복사해주세요.`를 표시한다.
- `.ics` 생성에 실패하면 선택창을 닫지 않고 오류를 표시해 다시 시도할 수 있게 한다.
- Google Calendar 링크는 유효한 데이터가 있을 때만 활성화한다.
- 캘린더 동작 버튼은 처리 중 연속 입력을 막고 완료 또는 실패 후 다시 활성화한다.
- 날짜가 유효하지 않거나 종료가 시작보다 늦지 않으면 개발 단계 테스트와 런타임 가드에서 실패 처리한다.

## 접근성

- 날짜와 시간은 화면에 자연스러운 한국어로 표시하고 `time` 요소에 기계 판독 가능한 값을 제공한다.
- 아이콘 버튼에는 동작을 설명하는 접근성 이름을 제공한다.
- 모달 선택창은 제목을 가진 대화상자로 노출한다.
- 닫기, `Escape`, 배경 선택 시 동일한 닫기 동작과 포커스 복원을 적용한다.
- 복사 결과는 시각 표시와 `aria-live` 알림을 함께 제공한다.
- 모든 버튼은 키보드 초점이 명확히 보여야 한다.

## 검증

### 공통 데이터 테스트

- 이름, 일시, 시간대, 장소, 홀, 주소가 확정값과 일치한다.
- 종료 시각은 시작 시각보다 정확히 90분 늦다.
- 기존 예식 안내 문구가 공통 데이터와 일치한다.

### 유틸리티 테스트

- `.ics`의 시작·종료 UTC 값, 제목, 장소, 이스케이프, CRLF가 올바르다.
- Google Calendar URL을 디코딩했을 때 동일한 일정 정보가 나온다.
- 일정 복사 문구와 주소 복사 값이 확정 형식과 일치한다.
- 잘못된 날짜와 역전된 기간을 거부한다.

### 컴포넌트 테스트

- 입장 화면에 실제 이름, 날짜, 시작 시각, 장소와 홀이 표시된다.
- 상세 메뉴에 종료 시각과 전체 주소가 표시된다.
- 두 화면의 캘린더 저장 버튼이 동일한 선택창을 연다.
- 주소 및 일정 복사의 성공·실패 상태가 표시된다.
- 선택창을 닫으면 실행 버튼으로 포커스가 복원된다.
- 메뉴와 선택창 상호작용이 맵 이동으로 전달되지 않는다.

### 회귀 검증

- 캐릭터 선택과 정원 입장이 기존대로 동작한다.
- 여정 바로 이동, 조이스틱, 포털, 미니맵 동작이 유지된다.
- 전체 테스트, 타입 검사, 프로덕션 빌드가 통과한다.
- 모바일 실제 화면에서 텍스트 잘림, 화면 겹침, 스크롤 막힘을 확인한다.

## 완료 기준

- 입장 전과 입장 후 모두 동일한 예식 정보를 확인할 수 있다.
- 기본 캘린더, Google Calendar, 일정 복사 세 동작이 정확한 90분 일정을 사용한다.
- 주소 복사 성공과 실패를 사용자가 확인할 수 있다.
- 새 UI가 게임 입력을 유발하지 않는다.
- 자동 테스트와 모바일 브라우저 검증을 통과한다.
