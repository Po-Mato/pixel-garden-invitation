# 새벽빛 프리즘 픽셀 정원 설계

## 목표

현재 맵의 굵은 `30px` 반복 무늬와 큰 단색 블록을 제거하고, 밝고 몽환적인 새벽빛 프리즘 정원으로 재구성한다. 이동과 충돌에 사용하는 `30px` 논리 타일, 맵 좌표, 캐릭터 크기, 모바일 무스크롤 UI는 그대로 유지한다.

## 시각 원칙

- 맵의 시각 표현 단위는 `4px`, `6px`, `8px`를 중심으로 사용한다.
- `30px` 패턴은 이동 좌표에만 남기고 바닥과 길의 반복 무늬에는 사용하지 않는다.
- 검정에 가까운 굵은 외곽선은 클릭 가능한 카드와 포털에만 유지한다.
- 장식은 주변 색보다 한 단계 짙은 `1px` 또는 `2px` 색상 외곽선을 사용한다.
- 민트, 라일락, 살구, 아이보리, 연노랑, 청록을 여러 레이어로 겹쳐 한 가지 색이 지배하지 않게 한다.
- 반투명 광원, 이슬, 작은 꽃, 잎 그림자, 프리즘 조각으로 깊이를 만든다.
- 캐릭터와 행동 요소 주변은 시각 밀도를 낮춰 가독성을 유지한다.

## 공통 맵 레이어

1. **기저색:** 구역별 밝은 두세 가지 색의 부드러운 수직 변화
2. **미세 타일:** `6px` 또는 `8px` 간격의 잔디 결, 점묘, 대각선 디더링
3. **환경광:** 모서리와 랜드마크 주변의 진주빛·금빛 색면
4. **길:** `4px` 모자이크, 얇은 하이라이트, 꽃잎 가장자리
5. **장식:** 내부 픽셀 디테일을 갖춘 꽃, 나무, 물, 가구, 액자
6. **부유 효과:** 이슬빛, 프리즘 먼지, 작은 반짝임
7. **전경:** 화면 가장자리의 잎과 꽃 그림자

공통 레이어는 `world-map`, `world-path`, `world-decoration-layer`와 가상 요소로 구현한다. DOM을 대량으로 추가하지 않고 CSS 배경과 기존 장식 데이터를 활용한다.

## 구역별 표현

### 입구: 새벽 안개 정원

- 상단은 옅은 아쿠아 안개, 하단은 민트와 연두 잔디로 구성한다.
- 길은 아이보리·민트·라일락 유리 모자이크처럼 표현한다.
- 연못에는 4px 물결, 수련 잎, 분홍 꽃, 하늘 반사광을 추가한다.
- 꽃 아치는 여러 크기의 작은 꽃과 잎이 이어지는 형태로 세분화한다.
- 나무는 세 가지 잎색과 작은 빛점을 사용해 단색 사각형처럼 보이지 않게 한다.

### 예식장: 진주빛 버진로드

- 잔디는 연두, 민트, 아이보리 이슬빛을 섞는다.
- 버진로드는 살구·아이보리·연노랑 모자이크와 진주 하이라이트를 사용한다.
- 길 가장자리에 작은 꽃잎과 금빛 점을 촘촘하게 배치한다.
- 장미 기둥과 버진로드 꽃장식은 줄기, 잎, 꽃송이를 4~6px 블록으로 나눈다.
- 분수는 물결과 수면 반짝임을 여러 겹으로 표현한다.

### 갤러리: 라일락 프리즘 전시

- 상단은 라일락·연보라, 하단은 세이지·민트로 전환한다.
- 길은 밝은 라일락 유리 조각과 아이보리 테두리로 표현한다.
- 액자 내부는 서로 다른 하늘, 잔디, 꽃 색면을 사용하고 작은 금빛 명판을 추가한다.
- 별빛 가랜드, 전시 조명, 모자이크 별은 4px 발광점과 얇은 광륜을 가진다.
- 라벤더 화단은 여러 보라색만 쓰지 않고 분홍, 아이보리, 청록 꽃을 섞는다.

### 라운지: 살구빛 티파티

- 상단은 살구·복숭아, 하단은 민트·청록 잔디로 구성한다.
- 길은 연분홍·아이보리·민트 직조 패턴으로 표현한다.
- 티 테이블에 찻잔, 접시, 케이크, 꽃병을 작은 픽셀로 구분한다.
- 선물과 디저트 카트는 리본, 포장지, 진열대 색을 세분화한다.
- 파티 플래그와 조명은 밝은 다색 포인트를 사용하되 행동 요소보다 대비를 낮춘다.

## 장식 세부화

- 큰 단색 면에는 최소 세 가지 명도 또는 색상 단계를 사용한다.
- 나무, 화단, 토피어리에는 잎 하이라이트와 그림자 픽셀을 추가한다.
- 물 표현에는 수평 하이라이트와 반사 픽셀을 사용한다.
- 가구와 구조물은 검정 외곽선 대신 짙은 녹색, 자주색, 갈색 등 주변색 외곽선을 사용한다.
- 사진 액자와 안내성이 없는 장식은 굵은 테두리를 `2px` 이하로 줄인다.
- 정보 카드, 포털, 조작 버튼은 현재 굵은 외곽선을 유지해 행동 요소임을 구분한다.

## 모션과 성능

- 부유 입자와 물빛은 `steps()` 애니메이션으로 표현한다.
- 동시에 움직이는 장식 수를 제한하고 큰 `filter: blur()`는 사용하지 않는다.
- 광륜은 작은 `box-shadow`와 반투명 색면으로 구현한다.
- `prefers-reduced-motion`에서는 반짝임, 물결, 꽃잎, 프리즘 먼지 애니메이션을 모두 정지한다.
- 별도 대형 이미지나 외부 네트워크 자산은 추가하지 않는다.

## 유지 조건

- `30px` 논리 타일 이동과 충돌 판정을 변경하지 않는다.
- 맵 `390:720` 비율과 모바일 `100dvh` 무스크롤 구조를 유지한다.
- 기존 장식 좌표와 카드·NPC·포털 비충돌 조건을 유지한다.
- 월드 캐릭터의 외곽선과 `48x72px` 표시 규격을 유지한다.
- 메뉴, HUD, 포털, 카드의 텍스트 가독성을 유지한다.

## 검증 기준

- CSS 계약 테스트에서 구역 배경 반복 단위가 `8px` 이하인지 확인한다.
- 공통 맵에 환경광, 부유 효과, 전경의 세부 레이어가 존재하는지 확인한다.
- 모든 기존·신규 장식 종류가 세부 스타일을 갖는지 확인한다.
- 맵 장식의 굵은 `3px`, `4px`, `5px` 검정 외곽선이 제거됐는지 확인한다.
- 네 구역의 색상 구성이 서로 다르면서 공통 프리즘 팔레트를 공유하는지 확인한다.
- 장식과 카드·NPC·포털 겹침이 0건인지 유지한다.
- `320x568`, `360x740`, `390x844`, `430x932`에서 스크롤과 경계 이탈이 없어야 한다.
- 네 구역 모바일 스크린샷과 데스크톱 스크린샷을 직접 검토한다.
- 모션 감소 설정에서 모든 새 애니메이션이 정지해야 한다.
- `pnpm test`, `pnpm typecheck`, `pnpm build`가 모두 통과해야 한다.
