# 반응형 프리즘 인터랙션 설계

## 목표

정원 입장 후 플레이 화면에서 모바일 브라우저 주소창 높이 변화와 관계없이 바깥 문서 스크롤을 완전히 제거한다. 맵의 구역 탭, 정보 카드, 포털, 초대장 메뉴, 조작부는 기존 새벽빛 프리즘 정원과 어울리는 화려하지만 정돈된 픽셀 인터페이스로 개선한다.

## 원인과 반응형 구조

- 현재 `body`와 일반 앱 셸은 `100vh` 최소 높이를 사용하지만 플레이 셸은 `100dvh`를 사용한다.
- 모바일 브라우저에서 `100vh`가 현재 보이는 높이보다 클 수 있어 플레이 셸 바깥에 문서 스크롤 영역이 남는다.
- 플레이 중에는 `html`과 `body`를 `100dvh`에 고정하고 `overflow: hidden`, `overscroll-behavior: none`을 적용한다.
- `.app-shell--playing`은 `position: fixed; inset: 0`으로 뷰포트에 고정해 상위 요소의 최소 높이에 영향을 받지 않게 한다.
- 입장 화면은 캐릭터 선택 콘텐츠가 길기 때문에 기존 세로 스크롤을 유지한다.
- 맵은 HUD를 제외한 남은 영역 안에서 `390:720` 비율로만 축소하며 논리 좌표와 `30px` 이동 타일은 변경하지 않는다.

## 시각 방향

기존 A안인 새벽빛 프리즘 정원을 이어가는 `프리즘 글라스 픽셀` 방향을 사용한다. 장식적인 요소는 1~2px 색상 외곽선, 아이보리 하이라이트, 민트·라일락·살구의 작은 색면으로 구성한다. 클릭 가능한 요소는 배경과 구분되도록 선명한 어두운 핵심선 한 겹을 유지하되 기존의 무거운 검정 상자처럼 보이지 않게 한다.

## 인터랙션 구성

### 구역 탭

- 네 칸 세그먼트 구조와 한 줄 레이아웃을 유지한다.
- 각 버튼에 구역별 색상 변수를 부여하고 상단 프리즘 광택을 추가한다.
- 활성 탭은 색면, 안쪽 아이보리 선, 짧은 빛 번짐으로 표시한다.
- 눌림 상태는 1px 이동과 그림자 감소로 표현한다.

### 정보 카드

- 카드 종류별 색상 변수를 사용한다.
- 왼쪽 위에 작은 마름모 배지를 만들고 카드 상단에 유리 하이라이트를 추가한다.
- 제목과 행동 라벨의 위계를 분리하고 행동 라벨에는 작은 방향 표시를 붙인다.
- `:focus-visible`에서 두 겹의 아이보리·라일락 포커스 링을 표시한다.

### 포털

- 살구 단색 상자 대신 구역별 프리즘 모자이크와 밝은 안쪽 선을 사용한다.
- 오른쪽에 방향 화살표를 표시해 이동 기능임을 명확히 한다.
- 호버 가능 환경과 터치 눌림 상태 모두에서 광택과 그림자가 반응한다.

### 메뉴와 조작부

- 초대장 메뉴 버튼은 작은 십자 아이콘 배지와 아이보리 유리 면을 사용한다.
- 메뉴 시트는 라일락·민트 프리즘 테두리, 반투명 모자이크 배경, 색상별 바로가기 버튼으로 구성한다.
- 조이스틱은 기능과 크기를 유지하면서 링·방향 버튼·중심점의 명도와 색을 프리즘 팔레트로 정리한다.

## 접근성과 모션

- 모든 버튼은 최소 현재 터치 영역을 유지한다.
- 키보드 `:focus-visible`을 카드, 포털, 탭, 메뉴 버튼에 제공한다.
- 텍스트 대비는 밝은 면 위 짙은 갈색 또는 짙은 색면 위 아이보리를 사용한다.
- 새 반짝임 애니메이션은 추가하지 않고 기존 모션 감소 설정을 보존한다.

## 검증 기준

- 플레이 상태에서 `html`, `body`, `#root`, `.app-shell--playing`의 바깥 스크롤이 없어야 한다.
- `320x568`, `360x740`, `390x844`, `430x932`에서 `scrollWidth === innerWidth`, `scrollHeight === innerHeight`여야 한다.
- 세로·가로 방향 전환 후에도 같은 조건을 만족해야 한다.
- 맵 비율, 타일 이동, 충돌 좌표, 캐릭터 크기는 기존 테스트를 통과해야 한다.
- 탭, 카드, 포털, 메뉴, 조이스틱에 프리즘 시각 변수와 포커스 상태가 존재해야 한다.
- 실제 화면에서 텍스트 잘림, 조작부 교차, 클릭 요소 교차가 없어야 한다.
- 전체 테스트, 타입 검사, 프로덕션 빌드가 통과해야 한다.
