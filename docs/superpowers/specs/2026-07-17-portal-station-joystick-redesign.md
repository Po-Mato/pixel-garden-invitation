# 포털·지하철역·조이스틱 재설계

## 목표

- 모든 포털 원형 진을 배경 출입구의 시각적 중앙에 맞춘다.
- 포털 원형 진과 실제 맵 이동 인식 범위를 일치시킨다.
- 지하철역의 중복되고 깨져 보이는 개찰구를 하나의 일관된 개찰구 은행으로 교체한다.
- 기존 CSS 십자 조이스틱을 이미지 기반 웨딩 나침반 조이스틱으로 교체한다.
- 타일 보행, 포털 도착 후 전환, 미니맵, 메뉴 입력 차단 등 기존 동작은 유지한다.

## 문제 진단

### 포털

현재 포털 효과는 타일 보행 목적지인 `approach`를 그대로 중심점으로 사용한다. 타일 중심은 배경 문의 기하학적 중앙에서 주로 15px 어긋나므로, 여러 맵에서 원형 진이 입구 중앙보다 좌측 또는 아래에 보인다. 시각 효과와 인식 범위는 일치하지만 둘 다 잘못된 위치에 놓이는 구조다.

### 지하철역

지하철역 배경에는 이미 기둥 세 개와 연결 난간이 그려져 있다. 런타임에서 단일 개찰구 기둥 전경을 세 번 반복해 덮으면서 원근, 폭, 경계가 중복되고 개찰구 통로가 읽히지 않는다.

### 조이스틱

현재 조이스틱은 CSS 직사각형 두 개를 교차한 형태다. 맵과 포털의 세밀한 픽셀아트에 비해 단순하고, 모바일에서 손잡이 이동 거리가 베이스 크기에 비해 커 보인다.

## 포털 설계

### 데이터 구조

`WorldPortal`에 선택적 `visualCenter: Point`를 추가한다.

- `approach`: 클릭 자동 보행이 도착하는 타일 중심. 기존 타일 경로 계약을 유지한다.
- `visualCenter`: 원형 진, 인식 타원, 미니맵 표식의 중심. 생략하면 `approach`를 사용한다.

`portalEntryRect`와 `pointInPortalEntry`는 `visualCenter ?? approach`를 중심으로 사용한다. 포털 클릭 경로 탐색은 계속 `approach`를 목적지로 사용한다.

### 위치 보정

- 상·하단 출입구는 배경 문의 가로 중앙에 맞춰 시각 중심을 기본적으로 15px 오른쪽으로 보정한다.
- 좌·우 출입구는 배경 통로의 세로 중앙에 맞춰 시각 중심을 기본적으로 15px 위로 보정한다.
- 배경 구조가 비대칭인 포털은 실제 출입구 중심을 기준으로 개별 좌표를 지정한다.
- 포털 원형 진과 인식 타원은 모두 `80x34px`를 사용한다.
- 접근 타일은 보정된 인식 타원 안에 포함되어야 한다.

### 동작 보존

- 포털 클릭 시 캐릭터는 기존처럼 `approach`까지 타일 단위로 걸어간다.
- 캐릭터가 보정된 인식 타원에 들어온 뒤에만 도착 지연과 fade 전환을 시작한다.
- 조이스틱 접근도 같은 타원을 사용한다.
- 미니맵의 포털 표식도 `visualCenter`를 사용한다.

## 지하철역 설계

### 배경 편집

`subway-station/pixel-background-source.png`의 중앙 개찰구 기둥과 연결 난간만 제거하고 기존 바닥 타일을 복원한다. 좌측 계단, 우측 승강장, 벤치, 노선 띠, 안전선과 외곽 구조는 보존한다.

### 통합 개찰구 전경

기존 `ticket-gate-front.png` 반복 배치 대신 `ticket-gate-bank-front.png` 하나를 사용한다.

- 런타임 위치: `x=360`, `y=360`
- 런타임 크기: `240x120px`
- 깊이 기준: `depthY=480`
- 구성: 스테인리스 본체 세 개, 통과 가능한 두 개 통로, 유리 플랩, 초록 화살표, 카드 인식등
- 화풍: 현재 지하철역의 고해상도 픽셀아트와 같은 탑다운 원근
- 배경: 투명 PNG

충돌 영역은 기존 세 본체 사각형을 유지한다.

- `360,360,60,120`
- `450,360,60,120`
- `540,360,60,120`

두 30px 통로는 타일 경로로 통과 가능해야 한다.

## 조이스틱 설계

### 이미지 생성

먼저 하나의 생성 시트에 베이스와 손잡이를 서로 떨어뜨려 생성한다.

- 베이스: 진주 아이보리 원판, 샴페인 골드 이중 테두리, 네 방향 나침반 장식, 작은 로즈 보석
- 손잡이: 자개 질감의 둥근 로즈 스톤과 금색 테두리
- 스타일: 맵과 어울리는 정교한 고해상도 픽셀아트
- 배경: 단색 마젠타 크로마키
- 금지: 글자, 방향 문자, 그림자, 잘린 외곽

크로마키 제거 후 다음 에셋으로 분리한다.

- `client/public/assets/ui/joystick-wedding-compass-base.png`
- `client/public/assets/ui/joystick-wedding-compass-thumb.png`

### 렌더링과 입력

`VirtualJoystick`의 포인터 캡처, 방향 정규화, 키보드 입력과 비활성 로직은 유지한다. CSS 도형 대신 베이스와 손잡이 PNG를 렌더링한다.

- 데스크톱 베이스: `90x90px`
- 모바일 베이스: `72x72px`
- 데스크톱 손잡이 이동: `30px`
- 모바일 손잡이 이동: `22px`
- 비활성 상태: 위치와 크기는 고정하고 채도와 밝기만 낮춘다.
- 포커스 링과 접근성 이름은 유지한다.

## 테스트와 검증

### 자동 테스트

- 모든 포털의 시각 중심이 해당 출입구 중심 계약과 일치하는지 검사한다.
- 모든 `approach`가 해당 포털 인식 타원 안에 포함되는지 검사한다.
- 클릭과 조이스틱 포털 전환이 보정 후에도 도착 지연과 fade 단계를 유지하는지 검사한다.
- 미니맵 포털 표식이 `visualCenter`를 사용하는지 검사한다.
- 지하철역 개찰구 전경이 하나만 렌더링되고 두 통로의 경로 탐색이 가능한지 검사한다.
- 조이스틱이 두 이미지 에셋을 사용하고 모바일·데스크톱 이동 거리를 지키는지 검사한다.
- 맵 에셋 감사, 전체 테스트, 타입 검사와 프로덕션 빌드를 실행한다.

### 시각 검증

- 모바일과 데스크톱에서 모든 맵의 포털 원형 진과 출입구 중앙을 확인한다.
- 지하철역에서 개찰구 원근, 바닥 복원, 두 통로, 캐릭터 깊이 관계를 확인한다.
- 조이스틱의 중립·상·하·좌·우 상태에서 손잡이가 베이스 밖으로 벗어나지 않는지 확인한다.
- 조이스틱, 미니맵, 초대장 메뉴를 눌렀을 때 맵 클릭 이동이 발생하지 않는지 확인한다.

## 배포

검증을 모두 통과한 변경을 `main`에 커밋·푸시해 GitHub Pages 배포를 실행한다. 워크플로 성공 후 공개 URL에서 새 번들, 지하철역 에셋, 조이스틱 에셋 응답과 브라우저 오류 유무를 확인한다.
