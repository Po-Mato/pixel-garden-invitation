# 모바일 청첩장 오시는 길 강화 설계

## 목표

하객이 게임에 입장하기 전과 입장한 후 모두 예식장 위치와 교통 정보를 빠르게 확인하고, 사용하는 지도 서비스로 이동하거나 예식장에 바로 전화할 수 있게 한다. 입장 화면, 초대장 메뉴, 월드의 오시는 길 장소는 하나의 공통 장소 데이터와 하나의 하단 시트를 사용해 정보 불일치를 방지한다.

## 확정 장소 정보

- 장소: `MJ컨벤션`
- 홀: `5층 파티오볼룸`
- 주소: `경기 부천시 소사구 경인로 386`
- 지하철: `1호선·서해선 소사역 1번 출구에서 도보 약 3분`
- 주차: `주차 2시간 무료 · 약 500대 이상 주차 가능`
- 대표번호: `032-347-5500`
- 지도 서비스: 네이버지도, 카카오맵, Google 지도

주차와 대표번호는 웹 참고 정보 사용을 사용자가 승인한 값이다. 변동 가능성이 있으므로 구조화된 공통 데이터에서 관리하고, 향후 예식장 안내가 변경되면 한 곳만 수정한다.

## 범위

### 포함

- 공통 장소 데이터에 전화번호, 지하철 안내, 주차 안내 추가
- 재사용 가능한 오시는 길 하단 시트
- 입장 화면의 오시는 길 진입 버튼
- 초대장 메뉴의 오시는 길 진입 버튼
- 월드의 오시는 길 장소에서 공통 하단 시트 열기
- 네이버지도, 카카오맵, Google 지도 외부 링크
- 주소 복사와 복사 결과 안내
- 예식장 대표번호 표시와 전화 링크
- 모달 포커스, 게임 입력 차단, 모바일 반응형 처리
- 단위·컴포넌트·통합·브라우저 검증

### 제외

- 지도 SDK 또는 지도 iframe 삽입
- 앱 전용 커스텀 URL 스킴에만 의존하는 연결
- 버스 노선과 실시간 교통 정보
- 현재 위치 권한 요청
- 별도 길찾기 전용 전체 화면
- 서버 또는 데이터베이스 저장
- 맵, 포털, 조이스틱 이동 규칙 변경

## 공통 데이터 모델

`shared/src/content.ts`의 `invitationContent.event.venue`가 장소 정보의 단일 출처가 된다. 기존 장소명, 홀, 주소에 다음 구조화된 값을 추가한다.

- 표시용 대표번호
- 지하철 안내 문구
- 주차 안내 문구
- 지도 검색에 사용할 목적지 이름

입장 화면, 초대장 메뉴, 월드 장소에 문구나 전화번호를 중복 작성하지 않는다. `tel:` 링크는 대표번호에서 숫자만 추출해 만들고, 지도 URL과 표시 문구도 모두 공통 장소 데이터에서 파생한다.

## 구성요소

### DirectionsSheet

세 진입점이 함께 사용하는 포털 기반 하단 시트다. 기존 `BottomSheet`의 포커스 트랩, `Escape`, 배경 선택, 닫기 버튼, 포커스 복원 동작을 재사용한다.

표시 순서는 다음과 같다.

1. `MJ컨벤션 5층 파티오볼룸`
2. 전체 주소와 주소 복사 버튼
3. 네이버지도·카카오맵·Google 지도 버튼
4. 지하철 안내
5. 자가용·주차 안내
6. 대표번호와 전화 버튼

지도 서비스는 이름과 아이콘을 함께 표시해 사용자가 원하는 서비스를 빠르게 구분할 수 있게 한다. 주소 복사는 현재 예식 정보 UI의 브라우저 어댑터와 성공·실패 문구 패턴을 재사용한다.

### 지도 링크 유틸리티

UI와 브라우저 전역 객체에 의존하지 않는 순수 함수가 장소명과 주소를 받아 세 지도 서비스의 HTTPS URL을 생성한다.

- 네이버지도: 장소명과 주소를 목적지 검색어로 사용
- 카카오맵: 장소명과 주소를 목적지 검색어로 사용
- Google 지도: 주소가 설정된 길찾기 URL 사용

URL에는 UTF-8 목적지 값을 안전하게 인코딩한다. 일반 외부 링크로 렌더링하고 새 탭을 사용해 현재 게임 상태를 유지한다. 새 창을 여는 자바스크립트 호출이나 앱 전용 스킴만 사용하는 방식은 사용하지 않는다.

## 화면 통합

### 입장 화면

기존 예식 요약의 `캘린더 저장`과 함께 `오시는 길` 버튼을 제공한다. 두 버튼은 작은 모바일 화면에서도 같은 명령 그룹으로 인식되되, 문구가 잘리거나 캐릭터 선택 영역을 과도하게 아래로 밀지 않아야 한다.

### 초대장 메뉴

상세 예식 요약의 장소·주소 영역에 오시는 길 진입점을 제공한다. 초대장 메뉴 위에서 `DirectionsSheet`가 열릴 때는 기존 캘린더 선택창과 동일하게 하단 시트가 최상위 레이어가 되고, 배경 메뉴는 입력을 받지 않는다.

### 월드 오시는 길 장소

월드의 `오시는 길` 장소를 선택하면 일반 설명만 있는 `SpotModal` 대신 공통 `DirectionsSheet`를 연다. 시트를 열고 닫거나 지도·복사·전화 동작을 사용하는 동안 캐릭터 위치가 변경되면 안 된다.

## 상호작용과 접근성

- 지도 링크는 HTTPS 범용 링크와 `target="_blank"`, 안전한 `rel` 속성을 사용한다.
- 전화 버튼은 `tel:0323475500` 링크로 제공한다.
- 주소는 선택 가능한 텍스트로 유지한다.
- 주소 복사 성공 시 `주소를 복사했습니다.`를 표시한다.
- 주소 복사 실패 시 `복사하지 못했습니다. 주소를 길게 눌러 복사해주세요.`를 표시한다.
- 복사 결과는 시각 문구와 `aria-live`로 함께 알린다.
- 하단 시트는 제목을 가진 모달 대화상자로 노출한다.
- 열릴 때 첫 동작으로 포커스를 이동하고 닫을 때 실행 버튼으로 복원한다.
- 시트 내부에서 `Tab`과 `Shift+Tab` 포커스가 순환한다.
- 시트의 포인터·키보드 입력은 월드 이동으로 전달되지 않는다.

## 오류 처리

- 공통 장소 데이터가 비어 있거나 URL 생성이 실패하면 해당 지도 버튼만 비활성화하고 시트의 다른 동작은 유지한다.
- 클립보드 권한이 거부돼도 원문 주소를 계속 선택할 수 있게 한다.
- 전화번호 데이터가 유효하지 않으면 전화 버튼을 렌더링하지 않고 번호 표시만 유지한다.
- 외부 지도 서비스의 장애는 청첩장 내부 오류 상태로 전환하지 않는다. 사용자가 새 탭을 닫고 기존 청첩장을 계속 사용할 수 있어야 한다.

## 반응형 설계

- 320px 너비에서도 지도 서비스 버튼과 전화 버튼 문구가 잘리지 않아야 한다.
- 지도 서비스 세 개는 좁은 화면에서 안정적인 열 또는 두 줄 배치로 전환한다.
- 세로 화면에서 시트 본문이 길어지면 시트 내부만 스크롤하고 닫기 버튼은 접근 가능한 위치에 유지한다.
- 가로 화면에서는 시트 높이를 제한해 지도 버튼과 닫기 동작을 동시에 확인할 수 있게 한다.
- 전체 문서에 가로 스크롤을 만들지 않는다.

## 검증

### 공통 데이터 테스트

- 장소명, 홀, 주소, 대표번호, 지하철, 주차 문구가 확정값과 일치한다.
- 대표번호에서 파생한 `tel:` 링크가 `tel:0323475500`과 일치한다.

### 유틸리티 테스트

- 세 지도 URL이 올바른 HTTPS 호스트를 사용한다.
- URL을 디코딩했을 때 장소명과 주소가 포함된다.
- Google 지도 URL이 길찾기 동작과 목적지를 포함한다.
- 잘못된 장소 데이터는 명시적으로 거부한다.

### 컴포넌트·통합 테스트

- 하단 시트에 주소, 지도 세 개, 지하철, 주차, 전화 동작이 표시된다.
- 주소 복사 성공·실패 상태가 표시된다.
- 입장 화면, 초대장 메뉴, 월드 장소가 같은 하단 시트를 연다.
- 초대장 메뉴 위에서 하단 시트 레이어가 올바르다.
- `Escape`, 배경, 닫기 버튼으로 닫히고 포커스가 복원된다.
- 시트 조작이 캐릭터 이동으로 처리되지 않는다.

### 회귀·브라우저 검증

- 기존 캘린더 저장, 캐릭터 선택, 정원 입장, 초대장 메뉴가 유지된다.
- 여정 이동, 조이스틱, 포털, 미니맵 동작이 유지된다.
- 전체 테스트, 타입 검사, 프로덕션 빌드가 통과한다.
- 320px 모바일, iPhone 14, 좁은 가로 화면에서 잘림·겹침·가로 스크롤을 확인한다.
- 운영 배포 후 세 지도 링크, 전화 링크, 주소 복사 실패 안내, 게임 입력 차단을 실제 브라우저에서 확인한다.
- GitHub Pages 실행 성공과 CI 산출물·운영 번들 일치를 확인한다.

## 완료 기준

- 세 진입점에서 동일한 장소 정보를 확인할 수 있다.
- 네이버지도, 카카오맵, Google 지도 중 원하는 서비스를 선택할 수 있다.
- 주소 복사와 예식장 전화가 동작한다.
- 지하철과 주차 안내가 확정 문구로 표시된다.
- 새 UI가 캐릭터 이동이나 기존 모달 흐름을 방해하지 않는다.
- 자동 테스트와 모바일 실서비스 검증을 통과한다.
