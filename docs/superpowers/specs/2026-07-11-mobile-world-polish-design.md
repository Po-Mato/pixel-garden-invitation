# 모바일 월드 플레이 개선 설계

## 목표

모바일 청첩장 월드를 한 화면 안에서 편하게 조작할 수 있도록 재구성한다. 캐릭터 이동은 타일 단위를 유지하면서 길게 눌렀을 때의 반복 속도를 낮추고, 네 구역은 서로 구분되는 웨딩 페스티벌 장식과 깊이감을 갖춘다. 하객 캐릭터는 원본 픽셀을 훼손하지 않는 외곽 효과로 배경과 분리한다.

## 이동 방식

- 논리 타일 크기는 기존 `30px`를 유지한다.
- 방향 입력을 시작하면 즉시 정확히 한 타일 이동한다.
- 입력을 계속 유지하면 첫 이동 후 `300ms` 동안 대기한다.
- 이후 `240ms`마다 한 타일씩 반복 이동한다.
- 방향을 바꾸면 새 방향으로 한 타일을 즉시 이동하고 초기 지연을 다시 적용한다.
- 입력을 놓으면 반복 이동과 보행 애니메이션을 즉시 멈춘다.
- 맵 터치 이동도 타일 중심 좌표만 사용하며 한 번에 한 타일씩 목표로 접근한다.
- 모든 이동은 기존 경계 및 충돌 판정을 통과해야 한다.
- 원격 전송 주기는 이동 체감 속도와 분리하고, 위치 메시지는 타일 좌표만 전송한다.

## 모바일 전체 화면 구조

- 플레이 화면은 `100dvh`를 기준으로 고정하고 문서 세로 스크롤을 만들지 않는다.
- `phone-frame`과 게임 월드는 사용 가능한 높이를 전부 사용하는 세로 그리드로 구성한다.
- 상단에는 현재 구역명, 연결 상태, 네 구역 탭을 한 줄 중심의 압축된 헤더로 배치한다.
- 맵은 남은 공간을 차지하는 유동 영역이며 기존 `390:720` 논리 좌표계를 CSS 스케일로 맞춘다.
- 캐릭터, NPC, 장식, 충돌 좌표는 논리 좌표계를 유지해 화면 크기와 무관하게 같은 위치 관계를 가진다.
- 방향 조작기는 맵 왼쪽 아래에 반투명 오버레이로 배치한다.
- 오른쪽 아래에는 초대장 메뉴 버튼과 현재 구역의 주요 행동 버튼을 배치한다.
- 전체 초대장 바로가기는 메뉴 버튼으로 여는 바텀시트에 제공한다.
- 바텀시트 내부만 필요할 때 스크롤할 수 있으며 기본 플레이 화면은 스크롤되지 않는다.
- `320x568`, `360x740`, `390x844`, `430x932` 화면에서 핵심 조작부가 겹치거나 잘리지 않아야 한다.
- 모바일 브라우저의 안전 영역을 `env(safe-area-inset-*)`로 반영한다.

## 맵 시각 고급화

### 공통

- 각 구역은 배경 타일, 가장자리 식재, 길 테두리, 중간 높이 장식, 전경 장식의 다섯 깊이 레이어를 가진다.
- 장식은 기존 데이터 기반 좌표를 유지하되 시각 전용 장식 종류를 확장한다.
- 장식이 통행로, 포털, 정보 카드, NPC, 스폰 타일을 가리지 않도록 회귀 테스트로 보호한다.
- 작은 반짝임, 꽃잎, 조명 애니메이션은 픽셀 단위 `steps()`로 표현하고 `prefers-reduced-motion`에서 비활성화한다.
- 장식은 클릭 가능한 요소처럼 보이지 않도록 행동 요소보다 대비와 외곽선을 낮춘다.

### 입구

- 꽃 아치와 환영 가랜드를 중심 랜드마크로 강화한다.
- 연못에 수련, 물결 하이라이트, 작은 돌 테두리를 추가한다.
- 입구 가장자리에 꽃 울타리, 리본 기둥, 나비와 안내 표식을 배치한다.
- 민트, 잎색, 노랑, 코랄을 함께 사용해 밝은 환영 정원으로 만든다.

### 예식장

- 중앙 버진로드에 꽃잎 가장자리와 금빛 포인트를 추가한다.
- 장미 기둥, 조명 줄, 분수, 좌석 꽃 장식을 대칭적으로 강화한다.
- 상단 예식 안내와 신랑신부 주변은 배경 장식을 정돈해 정보 가독성을 확보한다.
- 초록, 장미색, 아이보리, 금색을 균형 있게 사용한다.

### 갤러리

- 액자마다 서로 다른 픽셀 사진 색면과 작은 명판을 표현한다.
- 라벤더 화단, 별빛 가랜드, 전시 조명을 층별로 배치한다.
- 길은 보랏빛 모자이크 패턴으로 구분하고 가장자리에 작은 별 포인트를 추가한다.
- 보라색만 지배하지 않도록 초록, 금색, 아이보리, 코랄을 함께 사용한다.

### 라운지

- 티 테이블에 찻잔, 디저트, 의자 형태를 추가한다.
- 선물 더미, 우편함, 디저트 카트에 리본과 진열 디테일을 강화한다.
- 조명 줄과 파티 플래그로 축하 공간의 밀도를 높인다.
- 살구색, 민트, 체리색, 아이보리와 잎색을 조합한다.

## 캐릭터 분리 효과

- 월드에 표시되는 하객 캐릭터에만 픽셀 외곽 효과를 적용한다.
- 밝은 1픽셀 하이라이트와 어두운 2픽셀 그림자를 `drop-shadow()` 조합으로 만든다.
- 원본 PNG나 월드용 시트를 다시 가공하지 않아 방향별 프레임과 얼굴 형태를 보존한다.
- 캐릭터 선택 미리보기에는 외곽 효과를 적용하지 않는다.
- 신랑신부 NPC는 기존 그림자를 유지하되 하객과 비슷한 대비 수준으로 조정한다.
- 닉네임은 캐릭터 아래에 유지하고 작은 화면에서도 다른 조작부와 겹치지 않게 한다.

## 컴포넌트 변경

- `GameWorld`는 입력 유지 시간을 추적하는 타일 반복 제어기를 사용한다.
- 이동 시간 상수는 별도 모듈로 옮겨 단위 테스트에서 직접 검증할 수 있게 한다.
- `GameWorld`의 상단 구역 정보와 하단 바로가기는 모바일 게임 셸 구조로 재배치한다.
- 초대장 바로가기 목록은 메뉴 버튼과 바텀시트 열림 상태로 제어한다.
- `WorldDecoration`은 새 장식 종류도 기존 데이터 렌더링 계약으로 처리한다.
- 데스크톱에서는 최대 `430px` 휴대폰 프레임을 유지하고 화면 중앙에 배치한다.

## 접근성 및 오류 처리

- 메뉴, 방향 조작, 구역 탭은 명확한 접근성 이름과 포커스 표시를 가진다.
- 메뉴 바텀시트가 열리면 닫기 버튼과 배경 클릭으로 닫을 수 있다.
- 키보드 방향키는 페이지 스크롤을 유발하지 않고 게임 이동에만 사용한다.
- 모션 감소 설정에서는 장식 애니메이션과 불필요한 캐릭터 전환을 제거한다.
- 극단적으로 낮은 화면에서는 맵이 우선 축소되고 버튼과 텍스트는 최소 터치 크기를 유지한다.

## 검증 기준

- 이동 제어기 테스트에서 첫 입력 즉시 한 칸, `300ms` 초기 지연, `240ms` 반복 간격을 검증한다.
- 길게 누르는 동안 위치가 항상 `30px` 타일 중심에 정렬되는지 검증한다.
- 방향 전환과 입력 해제 시 반복 타이머가 초기화되는지 검증한다.
- 네 구역이 확장된 장식 종류와 시각 레이어를 갖는지 데이터 및 CSS 계약 테스트로 확인한다.
- 월드 캐릭터에만 외곽 효과가 적용되는지 컴포넌트와 CSS 테스트로 확인한다.
- 네 모바일 뷰포트에서 `scrollHeight <= clientHeight`, 가로 넘침 없음, 조작부 경계 이탈 없음으로 확인한다.
- 네 구역을 각 모바일 뷰포트에서 캡처해 장식, 카드, NPC, 캐릭터와 조작부 겹침을 확인한다.
- 데스크톱 화면과 모션 감소 설정도 별도로 확인한다.
- `pnpm test`, `pnpm typecheck`, `pnpm build`가 모두 통과해야 한다.
