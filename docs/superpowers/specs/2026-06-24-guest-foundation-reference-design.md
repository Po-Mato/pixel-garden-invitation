## 확정 기준

하객 캐릭터 재작업은 기존 생성 파츠를 기준으로 이어가지 않는다. 기존 결과는 파츠 시스템 검증용으로는 의미가 있지만, 사용자가 요구한 “예쁘고 멋진 하객 캐릭터” 기준에는 미달한다. 따라서 구현 기준은 새로 확정한 하객 기초 이미지로 고정한다.

기준 파일:

- 분위기 기준: `character-assets/reference/guest-foundation-concept-reference-v1.png`
- 구현 기준: `character-assets/reference/guest-foundation-sprite-reference-v1.png`

구현에서는 `guest-foundation-sprite-reference-v1.png`를 1차 기준으로 사용한다. `guest-foundation-concept-reference-v1.png`는 분위기, 패션 레이어, 웨딩 하객 정서를 확인하는 보조 기준이다.

## 기준 이미지의 역할

이번 기준 이미지는 최종 게임 에셋이 아니라, 하객 캐릭터를 다시 만들기 위한 아트 방향 원본이다. 구현할 때 이미지를 그대로 축소하거나 자동 변환하지 않는다. 대신 다음 요소를 사람이 읽히는 픽셀 스프라이트 규칙으로 해석한다.

- 둥글고 예쁜 얼굴
- 중앙 얼굴을 가리지 않는 헤어
- 작지만 분리된 눈, 작은 코, 부드러운 미소, 작은 볼
- 웨딩 하객으로 읽히는 단정한 포멀 의상
- 여성/남성 모두 사람다운 어깨, 팔, 손, 다리 비율
- 헤어, 의상, 액세서리를 분리 가능한 파츠처럼 볼 수 있는 구조

## 시각 목표

하객 캐릭터는 “배경 NPC”가 아니라 사용자가 조합해서 고르는 캐릭터다. 기본 원형만 봐도 예쁘고 멋져야 하며, 헤어와 의상은 그 원형을 꾸미는 역할이어야 한다.

목표 인상:

- 결혼식에 온 단정하고 매력적인 하객
- 신랑/신부보다 덜 화려하지만 같은 세계관에 있는 캐릭터
- 모바일 화면에서 작게 보아도 사람 얼굴과 포멀 의상이 읽히는 캐릭터
- 커스터마이저 확대 화면에서 얼굴과 의상 디테일이 납득되는 캐릭터

금지 인상:

- 외계인 같은 머리와 얼굴
- 눌리거나 찌그러진 얼굴
- 큰 검은 마스크처럼 보이는 눈
- 얼굴 중앙을 덮는 앞머리
- 막대팔, 사각 몸통, 저가형 블록 캐릭터
- 기준 이미지의 분위기 없이 파츠 수만 많은 캐릭터

## 구현 우선순위

### 1. base 원형부터 다시 만든다

기존 파츠를 부분 수정하는 방식으로는 기준 이미지 수준에 도달하기 어렵다. 먼저 `base`의 정면 원형을 만든다.

base가 책임질 것:

- 얼굴 외곽
- 눈, 코, 입, 볼
- 귀
- 목
- 기준 몸통
- 팔, 손, 다리, 발

base는 헤어와 의상이 없어도 “사람다운 하객 원형”으로 읽혀야 한다.

### 2. front-hair는 얼굴을 꾸밀 뿐 가리지 않는다

front-hair는 정수리, 앞머리 방향, 사이드락 일부만 담당한다. 얼굴 중앙 보호 영역을 침범해서 표정을 숨기면 실패다.

고정 감사 기준:

- 영역: `x=37`, `y=31`, `width=22`, `height=15`
- 허용 불투명 픽셀: 최대 `16`

이 기준은 “머리카락이 눈, 코, 입 중앙을 만들거나 덮는 방식”을 막기 위한 최소 안전장치다. 기준 이미지처럼 이마와 눈썹 위를 덮는 앞머리는 허용한다.

### 3. 기준 이미지의 눈 크기를 그대로 복제하지 않는다

기준 이미지는 방향 설정용이라 눈이 실제 96x144 게임 스프라이트보다 크다. 구현에서는 F1 얼굴 기준을 유지한다.

구현 규칙:

- 눈은 좌우가 분리된 작은 픽셀 덩어리여야 한다.
- 눈 사이에는 피부색 공간이 남아야 한다.
- 눈 밑 검은 픽셀이나 마스크형 표현은 금지한다.
- 코와 입은 눈보다 작고 가볍게 처리한다.

### 4. 의상은 파츠 교체가 가능한 구조로 만든다

기준 이미지의 의상은 스타일 목표를 제공한다. 구현에서는 다음처럼 파츠화 가능한 형태로 분해한다.

- 드레스: 상의, 허리선, 스커트 폭, 치맛단
- 한복: 저고리, 여밈, 치마/바지 폭, 장식 포인트
- 수트: 라펠, 셔츠, 타이, 바지선, 구두
- 블레이저: 재킷, 이너, 벨트/허리선, 슬랙스
- 액세서리: 귀걸이, 목걸이, 브로치, 가방, 넥웨어

## 작업 범위

이번 기준 확정 이후의 구현 범위:

- `scripts/author-guest-part-sources.mjs`의 base 원형 재작성
- base 정면 기준 프레임 리뷰 이미지 생성
- base가 승인된 뒤 헤어, 의상, 액세서리 순서로 파츠 재작성
- `character-assets/source/base/*`부터 재생성
- 이후 `hair`, `outfits`, `accessories`를 단계적으로 재생성
- 하객 아트 방향 문서에 확정 기준 이미지 경로와 규칙 반영

비범위:

- 신랑/신부 NPC 원본 변경
- 하객 프레임 규격 변경
- 런타임 레이어 구조 변경
- 기준 이미지를 자동 축소해서 소스 PNG로 쓰는 작업
- 커스터마이저 UI 구조 변경

## 검토 순서

구현은 한 번에 모든 파츠를 바꾸지 않는다. 기준이 다시 무너지지 않도록 다음 순서로 검토한다.

1. base 정면 원형만 만든다.
2. base 4x 확대 리뷰 이미지를 확인한다.
3. base가 기준 이미지의 얼굴/몸 비율에 충분히 가까운지 판단한다.
4. base 확정 후 헤어를 얹는다.
5. 헤어가 얼굴 중앙을 가리지 않는지 감사와 시각 검토를 함께 한다.
6. 의상과 액세서리를 얹는다.
7. 전체 카탈로그 컨택트 시트로 조합 결과를 검토한다.

## 성공 기준

작업 성공 기준:

- 하객 base만 봐도 얼굴이 예쁘고 사람답다.
- 얼굴이 눌리거나 삼각형으로 찌그러져 보이지 않는다.
- 헤어를 얹어도 눈, 코, 입, 볼이 분리되어 읽힌다.
- 기준 이미지의 여성 롱웨이브, 여성 한복, 남성 수트, 남성 미디엄헤어 인상이 파츠 조합으로 재현 가능하다.
- 96x144 원본과 4x 확대 리뷰 양쪽에서 모두 하객 캐릭터로 읽힌다.
- 감사 도구가 얼굴 보호 영역, 프레임 점유, 스타일 차이를 계속 검증한다.
- 모든 문서와 기준 설명은 한국어로 유지한다.
