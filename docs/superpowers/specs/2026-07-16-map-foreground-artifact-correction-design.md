# 맵 전경 아티팩트 보정 설계

## 목표

10개 맵의 배경과 전경 오브젝트를 함께 검수하고, 형태가 찌그러졌거나 배경과 어긋난 나무·개찰구·안내데스크를 같은 픽셀 화풍과 안정된 앵커로 보정한다.

## 범위

- `neighborhood`: 수관의 과도한 밝기와 둥근 외곽을 완화하고 화단의 그루터기 중심에 맞춘다.
- `subway-station`: 금속 슬래브처럼 보이는 개찰구 전면을 배경의 개찰구 폭·재질·원근에 맞춘다.
- `lobby`: 안내데스크 전면의 폭, 곡률, 상판 연결부와 색조를 배경에 포함된 상판에 맞춘다.
- 나머지 7개 맵의 전경 오브젝트는 형태와 정렬이 정상인 경우 그대로 유지한다.
- 맵 크기, 충돌 영역, 포털, 스폰 위치, 미니맵, 캐릭터 이동 및 깊이 가림 동작은 변경하지 않는다.

## 자산 처리

보정 대상은 기존 전경 원본 PNG를 편집 대상으로 사용한다. 배경의 해당 영역은 스타일과 배치 기준 이미지로 사용하며, 결과는 투명 배경의 단일 오브젝트로 정리한다. 최종 앱 자산은 기존 매니페스트 크기로 nearest-neighbor 리사이즈하여 교체한다.

원본과 출력 파일은 다음 계약을 유지한다.

- 원본: `map-assets/reference/v2/<zone>/*-source.png`
- 앱 출력: `client/public/assets/maps/v2/<zone>/*.png`
- 출력 크기와 파일명: `map-assets/reference/v2/manifest.json` 유지
- 투명 영역과 가시 알파 픽셀 모두 필수

## 배치 보정

오브젝트의 가시 알파 중심과 배경 본체의 중심을 기준으로 `client/src/game/world.ts`의 `x`, `y`, `width`, `height`, `depthY`를 확인한다. 자산 내부 여백으로 발생한 오차만 최소 조정하고, 충돌·포털 계약은 유지한다.

## 검증

1. 맵 자산 빌드와 감사 테스트를 통과시킨다.
2. 보정 전후 전체 맵 합성 시트를 생성해 10개 맵을 한 화면에서 비교한다.
3. 모바일 뷰포트에서 동네, 역사, 로비를 직접 확인한다.
4. 클라이언트 월드 테스트, 타입 검사, 프로덕션 빌드를 실행한다.
5. 사용자 요청 전에는 커밋하거나 푸시하지 않는다.
