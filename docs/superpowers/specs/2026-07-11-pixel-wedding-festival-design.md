# 픽셀 웨딩 페스티벌 맵 디자인

## 목표

기존 네 구역을 서로 다른 분위기의 화려하고 아기자기한 웨딩 페스티벌 정원으로 강화한다. 하객 캐릭터는 월드에서 표시 크기 `48×72px`를 유지하되 내부 렌더 해상도를 `24×36px`로 낮추고 최근접 보간으로 2배 확대해 픽셀 블록을 굵게 만든다.

## 시각 방향

- 전체 방향은 밝은 낮 시간대의 웨딩 페스티벌이다.
- 입구는 꽃 아치, 환영 리본, 연못, 나비가 중심이다.
- 예식장은 장미 통로, 분수, 꽃잎, 촛불과 웨딩 가랜드가 중심이다.
- 갤러리는 라벤더 정원, 전시 조명, 별 장식과 액자가 중심이다.
- 라운지는 티파티, 디저트 카트, 선물 상자, 우편함과 화분이 중심이다.
- 장식은 이동 경로와 상호작용 버튼을 가리지 않으며 화면 가장자리와 기존 빈 공간을 우선 사용한다.
- 보라색이나 베이지색 한 계열로 화면을 지배하지 않고 초록, 장미색, 노랑, 하늘색, 라벤더를 구역별로 배분한다.

## 맵 구조

`WorldDecorationKind`에 꽃 아치, 꽃잎, 나비, 토피어리, 조명 줄, 장미 기둥, 선물 더미, 디저트 카트, 별 장식 종류를 추가한다. 각 구역은 최소 12개의 명시적 장식과 최소 6개의 장식 종류를 갖는다. 장식은 기존 `WorldDecoration` 좌표 계약을 그대로 사용하며 충돌 영역에는 포함하지 않는다.

구역별 배경은 같은 잔디 그라디언트를 재색칠하는 방식 대신 서로 다른 타일 패턴과 가장자리 프레임을 사용한다. 길은 구역별 색과 테두리를 갖되 통행 가능 영역의 위치와 크기는 변경하지 않는다. 장식 애니메이션은 `steps()` 기반의 물결, 반짝임, 나비 날갯짓, 꽃잎 흔들림만 사용하고 `prefers-reduced-motion`에서는 비활성화한다.

## 캐릭터 렌더링

기존 원본 프리셋 시트 `96×144px` 프레임은 캐릭터 선택 화면용으로 유지한다. 빌드 시 각 프레임을 `24×36px`로 최근접 축소한 월드 전용 시트를 별도 생성한다.

- 걷기 시트: `72×144px`, 프레임 `24×36px`, 3열×4행
- 대기 시트: `48×36px`, 프레임 `24×36px`, 2열×1행
- 월드 표시: `48×72px`, 정확히 2배 확대
- 선택 화면 표시: 기존 `96×144px` 원본 사용

`CharacterSprite`는 `displayMode`에 따라 URL과 source size를 선택한다. 월드 캐릭터의 외부 레이아웃 크기, 닉네임 위치, 이동 좌표와 충돌 판정은 변경하지 않는다.

## 검증

- 자산 생성 테스트에서 월드 시트 개수와 정확한 치수를 검증한다.
- 자산 해석 테스트에서 월드와 미리보기 URL·source size가 분리되는지 검증한다.
- 맵 데이터 테스트에서 구역별 장식 수, 종류 수, 고유 테마 장식을 검증한다.
- 컴포넌트 테스트에서 구역별 테마 속성과 월드 전용 캐릭터 자산을 검증한다.
- 390×844 모바일과 1280×900 데스크톱에서 네 구역을 각각 촬영한다.
- 캐릭터 표시 영역이 `48×72px`이고 내부 이미지가 `24×36px`에서 2배 확대되는지 계산 스타일과 화면 픽셀로 확인한다.
- 장식, 버튼, 캐릭터, 닉네임이 겹치거나 화면 밖으로 넘치지 않는지 확인한다.
