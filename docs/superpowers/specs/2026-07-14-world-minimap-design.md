# 월드 미니맵 설계

## 목표

10개 맵의 전체 구조와 현재 캐릭터 위치를 이동 중에 계속 확인할 수 있는 표시 전용 미니맵을 제공한다.

## 배치와 크기

- 미니맵은 맵 뷰포트 우측 상단에 항상 표시한다.
- 일반 맵은 최대 `96x96px`, 세로로 긴 예식홀은 최대 `72x120px`에 맞춘다.
- 원본 맵의 종횡비를 유지하며 맵 안쪽으로 주요 요소를 단순화해 그린다.
- 미니맵은 조이스틱, 포털, 메뉴 버튼을 가리지 않도록 높은 `z-index`와 고정된 안쪽 여백을 사용한다.

## 표시 요소

- 맵 전체 경계
- 이동 가능 경로
- 장애물
- 포털
- 주요 정보 장소
- 현재 캐릭터 위치와 방향
- 현재 뷰포트 범위
- 자동 보행 중인 목표 포털 강조

## 상호작용

- 미니맵은 표시 전용이며 클릭이나 터치로 캐릭터를 이동시키지 않는다.
- 포털을 통한 맵 전환 규칙과 기존 맵 내 클릭 이동 규칙을 변경하지 않는다.
- 자동 보행이 시작되면 해당 포털을 미니맵에서 시각적으로 강조한다.

## 구조

- `client/src/game/minimap.ts`는 월드 좌표를 미니맵 좌표로 변환하는 순수 함수를 제공한다.
- `client/src/components/WorldMiniMap.tsx`는 활성 구역 데이터, 위치, 방향, 카메라, 목표 포털을 받아 SVG 미니맵을 렌더링한다.
- `GameWorld.tsx`는 기존 상태를 미니맵에 전달하고 맵 뷰포트 오버레이로 배치한다.
- CSS는 테마별 팔레트보다 가독성을 우선하는 중립 배경과 고대비 표시를 사용한다.

## 접근성

- 외곽은 `aside` 랜드마크와 `aria-label="현재 구역 미니맵"`을 사용한다.
- 시각 요소는 포커스 대상이 아니지만 클릭을 소비해 뒤쪽 맵으로 이동 명령이 전달되지 않게 한다.
- 현재 구역명은 미니맵 상단에 짧게 표시한다.

## 검증

- 일반, 가로형, 세로형 맵의 좌표 투영과 뷰포트 범위를 단위 테스트한다.
- 포털, 현재 위치, 뷰포트, 목표 포털이 렌더링되는지 컴포넌트 테스트한다.
- `320x568`, `390x844`, `430x932` 뷰포트에서 기존 HUD, 포털, 조이스틱과 겹치지 않는지 브라우저로 확인한다.
