# 타일 불꽃 포털 효과 설계

## 목표

- 가로·세로 포털에 동일한 사다리꼴 빛기둥을 사용해 형태가 어색한 문제를 제거한다.
- 포털 위치를 알려주는 세 개의 `30x30px` 타일 자체를 주 시각 효과로 강화한다.
- 작은 불꽃만 타일별로 짧게 상승시켜 몽환적인 움직임은 유지한다.
- 기존 3타일 판정, 최단 타일 자동 보행, 미니맵과 캐릭터 깊이 계약은 변경하지 않는다.

## 장식 구조

`GameWorld`의 포털 장식에서 다음 전역 요소를 제거한다.

- `.world-portal__beam--outer`
- `.world-portal__beam--core`
- `.world-portal__particle--one`부터 `.world-portal__particle--four`

포털 장식 DOM은 `.world-portal__effect` 안의 `.world-portal__tiles`와 세 개의 `.world-portal__tile`만 유지한다. 타일 중심 문양과 불꽃은 추가 DOM 없이 각 타일의 `::before`, `::after` 가상 요소로 구현한다.

## 타일 시각 효과

각 타일은 다음 레이어로 구성한다.

- 바닥: 청백색과 파란색이 교차하는 픽셀 격자
- 외곽: 밝은 이중 테두리와 타일 바깥으로 번지는 짧은 그림자
- 중심: `::before`의 작은 사각 룬과 내부 광점
- 불꽃: `::after`의 작은 픽셀과 `box-shadow` 복제로 만든 불꽃 묶음

불꽃은 타일 위로 최대 `24px`까지만 상승하며 위로 갈수록 투명해진다. 사다리꼴, 원뿔, 넓은 수직광과 타일 사이를 가로지르는 전역 입자는 사용하지 않는다.

세 타일은 같은 효과를 사용하되 `:nth-child(2)`, `:nth-child(3)`에 음수 animation delay를 적용해 동시에 깜빡이지 않게 한다. 가로 포털과 세로 포털 모두 타일 자체 좌표를 기준으로 애니메이션하므로 방향별로 별도 빛 폭을 계산하지 않는다.

목표 포털은 기존 `--portal-accent`, `--portal-core`, `--portal-deep`, `--portal-glow`의 금빛 변형을 그대로 사용한다. 따라서 새 타일과 불꽃도 추가 분기 없이 목표 색상으로 바뀐다.

## 깊이와 상호작용

- 포털 버튼 크기는 기존 `90x30px` 또는 `30x90px`를 유지한다.
- 세 타일의 판정과 클릭 영역은 변경하지 않는다.
- 포털 클릭 시 가장 가까운 진입 타일로 보행하는 동작을 유지한다.
- 포털 `z-index`는 `worldDepth(approach.y) - 100`을 유지해 캐릭터가 타일과 불꽃보다 앞에 표시된다.
- 포털 라벨과 미니맵 표식은 변경하지 않는다.

## 모션 축소

`prefers-reduced-motion: reduce`에서는 `.world-portal__tile`과 `.world-portal__tile::after` 애니메이션을 중지한다. 타일의 테두리, 중심 문양과 정적 외곽광은 유지해 이동 가능한 위치가 계속 보이게 한다.

## 테스트와 검증

### 자동 테스트

- 각 포털에 `.world-portal__tile`이 정확히 세 개 렌더링되는지 검사한다.
- `.world-portal__beam`과 `.world-portal__particle`이 렌더링되지 않는지 검사한다.
- 타일의 `30x30px` 크기, 이중 발광, 중심 문양과 `portal-tile-pulse`를 검사한다.
- `::after`가 `portal-tile-spark`를 사용하고 최대 `24px`만 상승하는지 검사한다.
- 세 타일에 서로 다른 animation delay가 적용되는지 검사한다.
- 모션 축소 규칙이 타일과 불꽃 애니메이션을 중지하는지 검사한다.
- 기존 월드, 경로 탐색, 포털 이동, 깊이, 미니맵과 조이스틱 테스트를 포함한 전체 테스트·타입 검사·빌드를 실행한다.

### 시각 검증

- 데스크톱과 모바일에서 가로 3타일 포털과 세로 3타일 포털을 각각 확인한다.
- 사다리꼴이나 넓은 수직광이 남아 있지 않은지 확인한다.
- 각 타일의 중심과 외곽이 분명하고 세 타일의 불꽃이 짧게 분산되어 움직이는지 확인한다.
- 캐릭터가 타일과 불꽃보다 앞에 표시되는지 확인한다.
- 포털 라벨, 미니맵, 조이스틱과 화면 UI가 겹치거나 이동 입력을 오염시키지 않는지 확인한다.

## 배포

검증을 통과한 변경을 `main`에 커밋하고 푸시한다. `.github/workflows/pages.yml`의 GitHub Pages 실행 성공 후 공개 URL에서 최신 CSS·JS 번들과 실제 포털 DOM을 다시 확인한다.
