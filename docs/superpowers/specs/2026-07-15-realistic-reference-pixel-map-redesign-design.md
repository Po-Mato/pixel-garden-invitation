# 사실적 레퍼런스 기반 픽셀 맵 전면 재설계

## 목표

현재 10개 맵의 단조로운 CSS 도형과 반복 무늬를 실제 공간처럼 구조와 재질이 읽히는 고밀도 픽셀 맵으로 교체한다. 각 맵은 사실적 공간 레퍼런스를 먼저 생성하고, 승인된 레퍼런스를 바탕으로 배경과 아티팩트를 픽셀 아트로 다시 제작한다.

이번 작업은 기존 좌표를 보존하는 시각 교체가 아니다. 맵 크기, 이동 가능 영역, 장애물, 상호작용 지점, NPC, 포털 위치를 새 공간 구성에 맞춰 함께 재설계한다. 기존 4방향 캐릭터, 타일 단위 이동, 포털 도착 후 전환, 페이드 효과, 미니맵, 초대장 콘텐츠는 유지한다.

## 승인된 결정

- 최종 앱에는 사진을 직접 사용하지 않고 고밀도 픽셀 아트만 사용한다.
- 각 맵의 사실적 레퍼런스 이미지를 먼저 만든 뒤 픽셀 배경과 아티팩트를 제작한다.
- 카메라는 정사영에 가까운 탑다운 3/4 구도를 사용한다.
- 현재 맵의 크기와 좌표를 고정하지 않고 공간 구조와 동선을 전면 재설계한다.
- 배경, 상호작용 오브젝트, 전경 가림 요소, 제한적인 광원 효과를 분리하는 레이어형 하이브리드 방식을 사용한다.

## 범위

대상 구역은 다음 10개다.

```text
home, neighborhood, subway-station, subway-train, venue-exterior,
lobby, bridal-room, ceremony-hall, restroom, banquet
```

각 구역에서 다음 항목을 교체하거나 재구성한다.

- 사실적 공간 레퍼런스
- 최종 픽셀 배경
- 핵심 아티팩트와 전경 오브젝트
- 맵 논리 크기와 카메라 안전 영역
- 이동 가능 영역과 장애물
- 상호작용 지점과 NPC 위치
- 포털 접근 지점, 도착 방향, 대상 맵 스폰 지점
- 미니맵에 투영되는 경로, 장애물, 포털, 정보 지점
- 장소별 제한적 환경 효과

## 범위 제외

- 하객 1번부터 12번까지의 캐릭터 원안과 픽셀 보행 이미지는 수정하지 않는다.
- 초대장 모달의 콘텐츠, 실시간 동기화 프로토콜, 방명록 데이터 구조는 변경하지 않는다.
- 대각선 이동, 자유 아날로그 이동, 미니맵 클릭 이동을 추가하지 않는다.
- 배경 이미지에 사람, 읽을 수 있는 간판 문구, UI 설명을 포함하지 않는다.
- 외부 이미지 URL이나 런타임 이미지 생성 서비스에 의존하지 않는다.

## 전체 미술 방향

하객의 이동 순서에 따라 공간의 축하 분위기와 시각 밀도가 점진적으로 커진다.

1. 집과 동네는 따뜻한 일상성과 출발의 기분을 전달한다.
2. 지하철은 시원하고 도시적인 재질과 이동감을 강조한다.
3. 예식장 외부와 로비는 자연광, 유리, 석재, 꽃 장식으로 목적지의 특별함을 드러낸다.
4. 신부 대기실과 예식홀은 인물과 버진로드가 시각적 중심이 되게 한다.
5. 연회장은 밝고 풍성한 마무리 공간으로 구성한다.

한 가지 색 계열이 맵 전체를 지배하지 않게 한다. 아이보리, 녹색, 코랄, 청록, 금속색, 목재색을 장소 재질에 맞춰 조합하고, 동일한 보라색·남색·베이지 변형만으로 화면을 채우지 않는다.

## 시점과 픽셀 규칙

- 탑다운 3/4 시점은 모든 맵과 오브젝트에서 같은 수직 기울기와 광원 방향을 공유한다.
- 바닥 평면과 벽 높이는 캐릭터가 가구보다 지나치게 크거나 작아 보이지 않도록 기존 캐릭터 표시 크기를 기준으로 정한다.
- 캐릭터 보행에 필요한 통로는 장식보다 높은 대비로 읽히되 바닥에 굵은 격자선을 표시하지 않는다.
- 최종 이미지는 픽셀 가장자리를 유지하고 브라우저에서 흐림 보간을 사용하지 않는다.
- 작은 오브젝트도 최소 세 단계의 명암과 재질 단서를 갖게 한다.
- 굵은 검정 외곽선은 포털처럼 행동 가능성이 중요한 요소에만 제한한다. 환경 오브젝트는 주변색 기반 외곽선을 사용한다.
- 빛과 반사는 형태를 가리지 않는 정적인 픽셀 또는 작은 `steps()` 애니메이션으로 표현한다.

## 구역별 시각 설계

### 1. 우리 집

- 아침 자연광이 들어오는 소형 아파트의 현관과 거실을 하나의 연속 공간으로 구성한다.
- 원목 바닥, 현관 타일, 낮은 소파, 작은 테이블, 신발장, 창가 식물로 생활감을 만든다.
- 주요 아티팩트는 창문, 소파, 테이블, 신발장, 현관문, 토피어리, 우편함, 스탠드 조명이다.
- 현관문까지의 이동 경로가 가구에 가리지 않고 분명하게 보이게 한다.

### 2. 동네 거리

- 봄꽃이 있는 주택가와 작은 상점이 이어지는 밝은 거리로 구성한다.
- 벽돌 보도, 차도, 횡단보도, 가로수, 카페 외벽과 화단을 서로 다른 재질로 표현한다.
- 주요 아티팩트는 나무, 가로등, 벤치, 횡단보도 표지, 화단, 지하철역 표지다.
- 도로와 보행로의 경계는 명확하지만 과도하게 굵은 타일선은 사용하지 않는다.

### 3. 지하철 역사

- 밝고 현대적인 대합실에서 개찰구를 지나 승강장으로 이어지는 구조를 사용한다.
- 유광 타일, 금속 개찰구, 노선색 띠, 안내판, 안전선에 서로 다른 반사광을 적용한다.
- 주요 아티팩트는 노선 안내, 개찰구 세트, 벤치, 천장 조명, 승강장 문이다.
- 출입 포털과 열차 탑승 포털이 동선상 혼동되지 않게 거리를 확보한다.

### 4. 지하철 차량

- 길게 이어지는 객차 내부와 창밖 도시 풍경을 함께 보여준다.
- 크림색 패널, 청록 좌석, 금속 손잡이, 출입문, 노선 표시 영역을 정교하게 구분한다.
- 주요 아티팩트는 도시 창문, 좌석 묶음, 손잡이, 천장 조명, 양쪽 출입문이다.
- 긴 가로 맵은 반복 패턴처럼 보이지 않도록 창밖 풍경과 좌석 소품에 변화를 준다.

### 5. 예식장 앞

- 유리와 밝은 석재를 조합한 가든 웨딩홀 정면과 진입 정원을 구성한다.
- 진입로, 꽃 아치, 작은 수경 요소, 화단, 건물 유리문이 중앙 입구로 시선을 유도한다.
- 주요 아티팩트는 웨딩홀 외벽, 유리문, 코랄 꽃 아치, 나무, 화단, 벤치, 조명이다.
- 건물 입구 포털은 장식과 겹치지 않고 캐릭터가 문 앞까지 실제로 걸어갈 수 있어야 한다.

### 6. 예식장 로비

- 밝은 대리석 바닥과 금속·유리 디테일을 가진 넓은 로비로 구성한다.
- 안내 데스크를 중심으로 외부, 신부 대기실, 화장실, 예식홀로 가는 방향이 공간 구조만으로 구분되게 한다.
- 주요 아티팩트는 안내 데스크, 축의대, 꽃 포토월, 소파, 대형 꽃 장식, 조명, 입구 표식이다.
- 분기 포털 사이에 충분한 보행 간격을 두고 서로의 자동 경로가 겹치지 않게 한다.

### 7. 신부 대기실

- 자연광, 아이보리 패브릭, 입체적인 꽃벽이 중심인 우아한 방으로 구성한다.
- 소파, 대형 거울, 화장대, 꽃 장식, 드레스 조명이 서로 다른 재질로 읽히게 한다.
- 주요 아티팩트는 꽃벽, 아이보리 소파, 화장대, 거울, 꽃병, 드레스 조명, 로비 문이다.
- 신부 NPC 앞 상호작용 공간과 출입문 복귀 동선을 분리한다.

### 8. 예식홀

- 높은 천장, 긴 버진로드, 대칭 좌석, 제단이 하나의 축으로 정렬된 세로형 공간으로 구성한다.
- 입구에서 제단이 보이되 긴 이동 중 좌석과 꽃 장식이 동일하게 반복돼 보이지 않도록 구간별 변화를 준다.
- 주요 아티팩트는 제단, 좌석 블록, 버진로드 꽃 장식, 촛불 또는 간접 조명, 신랑·신부 단상이다.
- 긴 맵의 이미지 밀도와 선명도를 유지하기 위해 필요한 경우 입구, 중앙 통로, 제단 구간을 별도 소스로 제작해 하나의 최종 배경으로 정합한다.

### 9. 화장실

- 밝은 테라조와 민트·화이트 타일을 사용한 깨끗한 공간으로 구성한다.
- 거울 조명, 세면대 금속, 칸막이, 식물을 재질 차이로 구분한다.
- 주요 아티팩트는 거울, 세면대, 칸막이, 식물, 로비 복귀문이다.
- 좁은 공간에서도 캐릭터가 세면대와 칸막이에 끼이지 않도록 중앙 통로를 확보한다.

### 10. 연회장

- 큰 창과 화사한 꽃 장식이 있는 밝은 피로연장으로 구성한다.
- 원형 테이블, 뷔페, 디저트 카트, 창가, 조명 그림자를 풍부하게 표현한다.
- 주요 아티팩트는 서로 다른 상차림의 원형 테이블, 뷔페, 가랜드, 디저트 카트, 예식홀 복귀문이다.
- 테이블 사이를 타일 단위로 이동할 수 있는 연속 통로와 방명록 상호작용 공간을 확보한다.

## 이미지 제작 파이프라인

각 맵은 한 단위씩 다음 순서로 완료한다. 다음 맵으로 넘어가기 전에 현재 맵의 원본과 앱 통합 결과를 모두 검수한다.

1. 인물과 읽을 수 있는 문구가 없는 사실적 탑다운 3/4 공간 레퍼런스를 생성한다.
2. 공간 구조, 시점, 조명, 핵심 오브젝트, 보행 통로, 출입구를 눈으로 검수한다.
3. 조건을 만족하지 않는 결과는 프로젝트에 저장하지 않고 같은 맵만 재생성한다.
4. 승인된 레퍼런스를 기반으로 픽셀 배경 원본을 생성한다.
5. 배경에서 분리해야 하는 상호작용 오브젝트와 전경 가림 요소를 동일한 시점과 광원으로 제작한다.
6. 원본 비율에 맞춰 최종 맵 크기를 정하고, 강제 비균등 확대나 축소 없이 앱용 자산을 만든다.
7. 픽셀 경계, 반복 흔적, 잘린 오브젝트, 원치 않는 인물과 문구가 없는지 다시 검수한다.
8. 게임에 통합한 뒤 실제 캐릭터 크기와 비교해 가구, 문, 통로의 스케일을 확인한다.

사실적 레퍼런스는 구도와 재질의 기준일 뿐 앱에 직접 노출하지 않는다. 생성 시트나 실패 결과를 앱의 정적 자산 폴더에 두지 않는다.

## 자산 구조

맵별 소스와 앱용 결과를 분리한다.

```text
map-assets/reference/v2/<zone-id>/
  realistic-reference.png
  pixel-background-source.png
  object-source-*.png

client/public/assets/maps/v2/<zone-id>/
  background.webp
  object-*.png
  foreground-*.png
```

- 레퍼런스와 픽셀 원본은 재생성 및 검수용 고해상도 파일로 유지한다.
- 앱 배경은 픽셀 경계를 보존하는 WebP 또는 PNG로 최적화한다.
- 투명도가 필요한 오브젝트와 전경은 PNG를 사용한다.
- 파일 이름은 구역 ID와 역할이 드러나는 영문 소문자 케밥 표기만 사용한다.
- 런타임에서는 로컬 정적 자산만 읽는다.

## 렌더링 레이어

맵 스테이지는 뒤에서 앞으로 다음 순서로 렌더링한다.

1. 장소별 폴백 바탕색
2. 맵 전체 바닥과 고정 건축물이 포함된 픽셀 배경
3. 캐릭터 뒤에 있어야 하는 배경 오브젝트
4. 포털, 상호작용 지점, NPC, 로컬 및 원격 캐릭터
5. Y좌표 기준으로 캐릭터와 앞뒤 관계가 바뀌는 오브젝트
6. 문틀, 나뭇가지, 샹들리에 하단 같은 전경 가림 요소
7. 입력을 받지 않는 제한적 광원과 환경 효과
8. 미니맵, HUD, 메뉴, 조이스틱

배경과 전경 이미지에는 `pointer-events: none`을 적용한다. 오브젝트는 시각 자산과 충돌 사각형을 분리해 꽃잎이나 그림자 같은 투명 여백 때문에 이동이 막히지 않게 한다.

## 월드 데이터 재설계

기존 `WorldZone`과 `WorldPortal` 데이터 모델은 유지하고 값을 새 레퍼런스에 맞춰 교체한다. 새로운 렌더링 자산 정보는 월드 데이터 또는 별도 시각 설정에서 구역 ID로 연결한다.

- 맵 논리 크기는 최종 배경 비율을 따르며 타일 크기의 정수 배수여야 한다.
- 스폰, 포털 접근점, 상호작용 지점은 타일 그리드 위에 놓는다.
- 보행 가능 영역과 장애물은 실제 바닥, 벽, 가구의 시각 경계와 일치시킨다.
- 모든 포털 접근점에는 상하좌우 타일 경로가 존재해야 한다.
- 포털 도착 방향은 문, 열차 출입구, 복도 방향과 일치시킨다.
- 대상 맵 스폰은 포털 또는 장애물과 겹치지 않고 카메라 안전 영역 안에 있어야 한다.
- 로비의 네 방향 분기와 예식홀의 긴 세로 동선은 자동 경로 탐색이 실패하지 않게 단일 연결 그래프를 유지한다.

## 미니맵 연동

미니맵은 배경 이미지를 축소 표시하지 않고 월드 데이터를 단순화해 그린다.

- 새 맵 경계 비율을 그대로 유지한다.
- 이동 가능 영역, 장애물, 포털, 정보 지점, 현재 캐릭터, 카메라 뷰포트를 표시한다.
- 실제 배경 구조와 데이터 경계가 어긋나지 않는지 맵별 검수 항목에 포함한다.
- 미니맵은 표시 전용이며 클릭 또는 터치가 월드 이동으로 전달되지 않는다.
- 자동 보행 중에는 목표 포털만 강조한다.

## 포털과 입력 동작

현재 포털 이동 규칙은 그대로 유지한다.

1. 포털 클릭 시 대상 접근점까지 타일 경로를 계산한다.
2. 캐릭터가 타일 단위로 포털 방향으로 이동한다.
3. 접근점에 도착하고 포털과 접촉한 상태가 확인된 뒤 짧은 대기 시간을 적용한다.
4. 페이드 아웃 후 맵과 스폰을 변경하고 페이드 인한다.

조이스틱으로 직접 포털에 접근한 경우에도 같은 도착 판정과 페이드 전환을 사용한다. 조이스틱, 초대장 메뉴, 미니맵을 조작한 이벤트는 월드 클릭 이동을 발생시키지 않는다.

## 모션과 성능

- 큰 배경에는 런타임 필터, 확대 블러, 복잡한 혼합 모드를 사용하지 않는다.
- 창문 빛, 전광판, 차량 창밖, 꽃잎처럼 장소 정체성에 필요한 요소만 작은 애니메이션으로 추가한다.
- 애니메이션은 `transform`과 `opacity`, 픽셀 단계형 프레임을 우선한다.
- `prefers-reduced-motion`에서는 환경 애니메이션을 정지하고 포털 전환은 즉시성이 떨어지지 않는 단순 페이드로 축소한다.
- 화면 밖 장식이 불필요하게 계속 애니메이션되지 않게 한다.
- 이미지 로딩 실패 시 장소별 폴백 배경색이 남아 빈 화면이 되지 않게 한다.

## 실패 처리

- 이미지 생성 결과가 다른 시점, 인물 포함, 읽을 수 있는 가짜 문구, 왜곡된 구조, 잘린 핵심 오브젝트를 포함하면 저장하지 않는다.
- 픽셀 변환 결과가 단순 필터처럼 흐리거나 캐릭터와 픽셀 밀도가 맞지 않으면 같은 맵만 재생성한다.
- 배경 비율과 맵 비율이 다르면 CSS로 찌그러뜨리지 않고 구도 또는 맵 크기를 수정한다.
- 오브젝트 투명 여백이 충돌을 넓히지 않게 시각 영역과 충돌 영역을 별도로 조정한다.
- 새 자산이 로드되지 않거나 디코딩되지 않는 경우 자동 테스트와 브라우저 콘솔 검사에서 실패로 처리한다.

## 맵 단위 검수

각 맵은 다음 항목을 모두 통과한 뒤 다음 맵을 제작한다.

- 사실적 레퍼런스가 승인된 시점, 공간 구조, 장소 정체성을 만족한다.
- 픽셀 배경에 인물, 읽을 수 있는 문구, 심한 반복, 형태 왜곡이 없다.
- 핵심 아티팩트가 서로 다른 재질과 형태로 구분된다.
- 캐릭터와 문, 가구, 좌석의 크기 관계가 자연스럽다.
- 모든 이동 가능 타일이 시각적 바닥 안에 있다.
- 장애물과 시각적 구조물이 일치한다.
- 포털 클릭과 조이스틱 접근이 모두 도착 후 전환된다.
- 캐릭터와 오브젝트의 앞뒤 가림이 자연스럽다.
- 미니맵의 경계, 경로, 장애물, 포털이 실제 맵과 일치한다.
- 모바일과 데스크톱에서 HUD, 미니맵, 조이스틱, 메뉴가 맵의 핵심 요소를 가리지 않는다.

## 전체 검증 기준

- 정확히 10개 구역에 새 픽셀 배경이 적용돼야 한다.
- 각 구역의 핵심 아티팩트가 새 자산 또는 새 배경 디테일로 교체돼야 한다.
- 집에서 연회장까지 모든 포털을 순서대로 통과할 수 있어야 한다.
- 역방향 포털과 로비 분기 경로가 모두 연결돼야 한다.
- 포털 클릭 직후 맵이 바뀌지 않고 도착·대기·페이드 순서를 지켜야 한다.
- 조이스틱으로 포털에 접근해도 동일한 전환이 실행돼야 한다.
- 미니맵, 조이스틱, 초대장 메뉴 클릭이 캐릭터 이동을 유발하지 않아야 한다.
- `320x568`, `360x740`, `390x844`, `430x932` 모바일 뷰포트와 데스크톱 뷰포트에서 10개 맵을 직접 검토해야 한다.
- 긴 지하철 차량과 예식홀에서도 배경이 흐려지거나 접합부가 드러나지 않아야 한다.
- 브라우저 콘솔에 자산 로딩, 디코딩, 좌표 계산 오류가 없어야 한다.
- 전체 테스트, 타입 검사, 프로덕션 빌드가 통과해야 한다.
- 배포 후 실제 GitHub Pages URL에서 모바일·데스크톱 전체 여정을 다시 확인해야 한다.
