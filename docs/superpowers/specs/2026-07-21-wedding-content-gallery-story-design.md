# 사진·소개·스토리 실사용화 설계

## 1. 목표

모바일 청첩장의 `신랑신부 정원`, `사진 갤러리`, `연애 스토리 꽃길`을 실제로 감상할 수 있는 콘텐츠 화면으로 완성한다. 현재 촬영 전 단계이므로 얼굴 식별을 최소화한 임시 웨딩 이미지와 사실을 지어내지 않는 중립 문구를 사용한다. 추후 실제 사진과 문구가 준비되면 UI 코드 변경 없이 정적 자산과 공통 콘텐츠 데이터만 교체할 수 있어야 한다.

## 2. 확정 범위

- 화사한 정원 웨딩 에디토리얼 임시 이미지 10장
- 이승재·이건희 개별 소개와 공동 초대 문구
- 4단계 세로 스토리 타임라인
- 에디토리얼 스크롤 갤러리
- 전체 화면 사진 감상, 모바일 스와이프와 데스크톱 키보드 지원
- 기존 맵 지점과 초대장 메뉴에서 같은 콘텐츠 패널 재사용
- 이미지 최적화, 접근성, 입력 차단, 회귀 테스트와 공개 Pages 배포

범위에서 제외한다.

- 실제 두 사람의 외모를 추정하거나 재현하는 이미지
- 실제 첫 만남 날짜·장소·개인적 사건의 창작
- 사진·문구 업로드용 CMS 또는 관리자 편집 화면
- Cloudflare R2 같은 원격 이미지 저장소

## 3. 정적 콘텐츠 구조

생성 원본은 교체와 재가공을 위해 배포 자산과 분리한다. 배포용 이미지는 `client/public/images/wedding-gallery/`에 번호순으로 저장하고, 공통 예식 데이터에 다음 구조를 추가한다.

- `coupleProfiles`: 역할, 이름, 짧은 소개, 작은 대표 이미지, 대체 텍스트
- `coupleMessage`: 두 사람의 공동 초대 문구
- `gallery`: 이미지 ID, 파일 경로, 폭·높이, 방향, 대체 텍스트, 선택적 캡션
- `storyTimeline`: 단계 ID, 제목, 중립 문구, 선택적 이미지 ID

파일명과 메타데이터가 표시 순서의 유일한 기준이 되며 실제 사진 교체 시 동일한 ID와 파일명을 유지할 수 있다. 이미지 데이터는 UI 컴포넌트에 직접 하드코딩하지 않는다.

## 4. 임시 이미지 세트

모든 이미지는 같은 정원, 흰 꽃, 밝은 자연광, 포멀 웨딩 의상으로 통일한다. 정면 얼굴 클로즈업을 피하고 뒷모습·옆모습·손·부케·실루엣을 중심으로 구성한다.

1. `01-cover`: 정원 통로에 선 커플의 가로 뒷모습, 대표 이미지
2. `02-dress-bouquet`: 드레스와 부케 중심 세로 컷
3. `03-side-walk`: 나란히 걷는 커플 옆모습 세로 컷
4. `04-bench-silhouette`: 정원 벤치의 커플 실루엣 세로 컷
5. `05-veil-flowers`: 베일과 흰 꽃 디테일 세로 컷
6. `06-hands-rings`: 손과 반지 디테일 가로 컷
7. `07-bouquet-still`: 부케 정물 세로 컷
8. `08-under-tree`: 나무 아래 커플의 가로 옆·뒷모습
9. `09-garden-aisle`: 사람이 작게 보이는 정원 통로 가로 전경
10. `10-sunlit-finale`: 햇살 속 커플 뒷모습 세로 컷

생성 원본을 직접 수정하지 않고 배포용 파생본을 만든다. 대표 이미지에는 우선 로딩을 적용하고 나머지는 지연 로드한다. 배포본은 모바일과 고밀도 화면을 위한 복수 해상도 WebP를 기본으로 하며, 각 이미지의 표시 비율을 메타데이터로 고정해 레이아웃 이동을 방지한다.

## 5. 임시 소개 문구

문구는 개인적 성격이나 실제 사건을 단정하지 않는다.

### 신랑 이승재

> 새로운 계절을 함께 시작합니다. 귀한 걸음에 감사드리며 기쁜 마음으로 기다리겠습니다.

### 신부 이건희

> 함께 걷는 첫날을 소중한 분들과 나누고 싶습니다. 따뜻한 축복으로 함께해 주세요.

### 두 사람의 인사

> 저희 두 사람의 새로운 시작에 함께해 주시면 더없는 기쁨이겠습니다.

개별 소개는 모바일에서 세로로 배치하고 이름, 역할, 작은 이미지, 두 문장 소개 순서로 읽힌다. 두 사람의 공동 문구는 개별 소개 뒤에 하나의 연결 문장으로 표시한다.

## 6. 4단계 스토리

실제 정보가 준비되면 문구만 교체할 수 있도록 단계 ID와 순서를 고정한다.

1. **첫 인사**  
   서로의 일상에 처음 인사를 건넨 순간부터 두 사람의 이야기가 시작되었습니다.
2. **함께한 시간**  
   크고 작은 계절을 함께 지나며 서로에게 가장 편안한 사람이 되었습니다.
3. **결혼을 약속한 마음**  
   앞으로의 모든 날도 같은 편으로 걷고 싶다는 마음으로 평생을 약속했습니다.
4. **우리의 결혼식**  
   이제 한 가족이 되는 첫날, 소중한 분들을 모시고 기쁨을 나누려 합니다.

타임라인은 한 개의 세로 기준선과 단계 표시, 제목, 짧은 본문으로 구성한다. 장식보다 가독성을 우선하며 기존 하단 시트 안에서 독립적인 페이지 섹션처럼 읽히게 한다.

## 7. 에디토리얼 갤러리

갤러리는 다음 리듬을 기본으로 한다.

1. 대표 가로 사진 전체 폭
2. 대표 사진 캡션
3. 세로 사진 두 장의 2열 묶음
4. 가로 사진 전체 폭
5. 세로 사진 두 장의 2열 묶음
6. 디테일과 마무리 사진을 전체 폭과 2열로 교차 배치

고정 높이로 사진을 찌그러뜨리지 않고 원본 비율과 의도된 크롭 위치를 사용한다. 모든 사진은 버튼 역할을 갖고 키보드와 터치로 열 수 있어야 한다.

## 8. 전체 화면 감상

사진을 선택하면 기존 하단 시트보다 위에 전체 화면 감상 레이어를 연다.

- 모바일: 좌우 스와이프로 이전·다음 이동
- 데스크톱: 이전·다음 아이콘 버튼과 방향키 이동
- 공통: 현재 번호, 전체 장수, 닫기 버튼, 선택적 캡션
- `Esc`로 닫기
- 처음과 마지막에서 순환하지 않고 경계를 명확히 표시
- 열 때 선택한 사진으로 시작하고 닫을 때 해당 사진 버튼으로 포커스 복귀
- 열린 동안 배경 스크롤, 지도 클릭, 조이스틱, 키보드 이동을 모두 차단
- `prefers-reduced-motion`에서는 전환 애니메이션 제거

확대 레이어는 사진을 화면에 맞게 보여주되 잘라내지 않는 감상 모드가 기본이다. 브라우저 기본 확대 제스처를 방해하는 별도 핀치 줌은 구현하지 않는다.

## 9. 기존 화면 연결

- `couple`: 신랑신부 소개 전용 콘텐츠
- `gallery`: 에디토리얼 갤러리와 전체 화면 감상
- `story`: 4단계 세로 타임라인

기존 맵 지점과 초대장 메뉴는 동일한 패널 컴포넌트를 연다. 패널이 열리면 기존 RSVP·오시는 길 화면과 같은 방식으로 월드 입력을 중지하고, 닫히면 열었던 버튼이나 지점으로 포커스를 복귀한다.

## 10. 오류 처리

- 이미지가 실패해도 선언된 비율의 영역을 유지한다.
- 실패 영역에는 중립 배경과 대체 설명을 표시한다.
- 전체 화면 이미지가 실패해도 닫기와 이전·다음 이동은 유지한다.
- 이미지 메타데이터 누락, 파일 누락, 중복 ID, 지원하지 않는 방향 값은 빌드 전에 실패시킨다.
- 실제 사진 교체 후 가로·세로 비율이 선언과 다르면 자산 감사에서 오류를 보고한다.

## 11. 접근성과 성능

- 모든 사진에 장면을 설명하는 한국어 대체 텍스트 제공
- 아이콘 버튼에 접근 가능한 이름과 툴팁 제공
- 확대 레이어에 포커스 고정과 포커스 복귀 적용
- 대표 이미지 우선 로드, 나머지 이미지 지연 로드
- `srcset`과 `sizes`로 화면에 필요한 크기만 전송
- 명시적 `width`, `height` 또는 `aspect-ratio`로 누적 레이아웃 이동 방지
- 작은 화면에서도 버튼과 캡션이 사진을 가리지 않게 안전 영역 반영

## 12. 검증과 배포

자동 검증:

- 갤러리 이미지 10장과 콘텐츠 메타데이터의 일치
- 이미지 ID·파일명·방향·폭·높이·대체 텍스트 검증
- 에디토리얼 배치 순서와 대표 이미지 우선 로드
- 전체 화면 열기·닫기·이전·다음·경계 처리
- 모바일 스와이프와 데스크톱 방향키·`Esc`
- 포커스 고정과 원래 사진으로의 복귀
- 갤러리·소개·스토리 중 지도·조이스틱 입력 차단
- 이미지 실패 시 레이아웃 및 탐색 유지

브라우저 검증:

- `390×844` 모바일 세로
- 가로 모바일
- `1440×1000` 데스크톱
- 사진 로딩 전후 가로 오버플로와 레이아웃 이동 확인
- 실제 맵 지점과 초대장 메뉴 양쪽에서 소개·갤러리·스토리 열기
- 전체 화면 감상 중 배경 입력 차단과 닫은 뒤 포커스 복귀
- 브라우저 콘솔과 네트워크 이미지 오류 확인

전체 테스트, 타입 검사, 프로덕션 빌드와 독립 리뷰를 통과한 뒤 `main`에 푸시한다. Worker 변경이 없으면 GitHub Pages만 배포하며, 공개 URL에서 최종 번들과 이미지 10장의 HTTP 응답 및 모바일·데스크톱 화면을 다시 확인한다.
