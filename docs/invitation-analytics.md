# 방문 통계·하객 행동 분석

## 운영 화면

- 관리자 URL: `?admin=analytics`
- 조회 기간: 최근 7일, 30일, 90일, 전체
- 제공 지표: 방문·재방문, 게임/간편 초대장 진입, 오시는 길·지도·전화, 공유·캘린더, RSVP 전환, 방명록, 갤러리, 로딩 시간, 클라이언트 오류
- CSV: 선택 기간의 일별 집계를 내려받는다.

## 개인정보 원칙

`invitation_analytics_daily`에는 날짜, 이벤트 이름, 허용된 분류, 횟수, 성능 합계만 저장한다. IP 주소, 사용자 식별자, 세션 식별자, URL 상세 경로, 오류 메시지는 저장하지 않는다. 재방문 여부는 브라우저 로컬 저장소의 방문 여부만으로 계산하며 서버로 식별자를 보내지 않는다.

## 집계 기준

- 방문: 페이지를 새로 연 횟수
- 재방문: 같은 기기의 로컬 방문 표시가 이미 있는 방문
- 모드 진입: 게임 정원 또는 간편 초대장을 실제로 연 횟수
- RSVP 전환: 답변 영역 노출, 입력 시작, 신규 답변 전송 완료
- RSVP·방명록 실제 건수: 선택 기간에 D1에 생성된 실제 레코드
- 로딩 시간: 브라우저 Navigation Timing의 전체 로딩 시간, 최대 60초로 제한
- 클라이언트 오류: 스크립트, 비동기 Promise, 리소스 오류 유형별 횟수만 집계

행동 지표는 기능 배포 이후부터 쌓인다. 기존 RSVP·방명록 레코드는 실제 건수에 포함되지만 과거 화면 조회나 클릭은 소급되지 않는다.

## 배포

1. `0012_invitation_analytics.sql`을 D1에 적용한다.
2. Worker를 배포한다.
3. Pages 클라이언트를 배포한다.
4. 공개 초대장에서 이벤트 수집 요청이 `204`인지 확인한다.
5. 관리자 통계 화면에서 기간 조회와 CSV 다운로드를 확인한다.
